# 04. Esercizi LAB06 - Utility statiche e input robusto

## Obiettivo degli esercizi

In questi esercizi applicherai in autonomia lo schema studiato nel LAB06:

```text
leggere -> convertire -> validare -> elaborare -> stampare
```

L'obiettivo è consolidare l'uso di:

- `Scanner`;
- `nextLine()`;
- `Integer.parseInt(...)`;
- `Double.parseDouble(...)`;
- `try/catch`;
- metodi statici;
- cicli di richiesta fino a input valido;
- separazione tra input, calcolo e output.

Gli esercizi devono essere svolti creando programmi separati, uno per ogni consegna.

---

## 1. Regole generali

Per tutti gli esercizi valgono queste regole:

1. crea un solo oggetto `Scanner` nel `main`;
2. usa `nextLine()` per leggere l'input;
3. converti i valori numerici con `Integer.parseInt(...)` o `Double.parseDouble(...)`;
4. gestisci gli errori di conversione con `try/catch`;
5. usa metodi statici per evitare codice duplicato;
6. non chiudere lo `Scanner` prima della fine del programma;
7. usa il punto come separatore decimale, ad esempio `12.5`;
8. non usare la virgola come separatore decimale, ad esempio `12,5`.

# Esercizio 1 - Lettura robusta di un intero

## Obiettivo

Scrivere un programma che chieda all'utente un numero intero e continui a richiederlo finché il valore inserito non è valido.

## File da creare

```text
Esercizio01_LeggiIntero.java
```

## Requisiti

Il programma deve contenere un metodo statico:

```java
public static int leggiIntero(Scanner scanner, String messaggio)
```

Il metodo deve:

- mostrare il messaggio ricevuto come parametro;
- leggere una riga con `nextLine()`;
- convertire la stringa con `Integer.parseInt(...)`;
- intercettare `NumberFormatException`;
- ripetere la richiesta finché il valore non è valido;
- restituire il numero intero valido.

## Codice di partenza

```java
import java.util.Scanner;

public class Esercizio01_LeggiIntero {

    public static int leggiIntero(Scanner scanner, String messaggio) {
        // completa il metodo
        return 0;
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int numero = leggiIntero(scanner, "Inserisci un numero intero: ");

        System.out.println("Numero acquisito: " + numero);

        scanner.close();
    }
}
```

## Test richiesti

Esegui almeno questi test:

| Input | Comportamento atteso |
|---|---|
| `25` | accettato |
| `abc` poi `10` | rifiuta `abc`, poi accetta `10` |
| `12.5` poi `7` | rifiuta `12.5`, poi accetta `7` |

# Esercizio 3 - Calcolo dell'area di un rettangolo

## Obiettivo

Scrivere un programma che calcoli l'area di un rettangolo.

## File da creare

```text
Esercizio03_AreaRettangolo.java
```

## Requisiti

Il programma deve contenere:

```java
public static double leggiDouble(Scanner scanner, String messaggio)
```

```java
public static double calcolaArea(double base, double altezza)
```

```java
public static void stampaRisultato(double base, double altezza, double area)
```

Il programma deve:

- chiedere la base;
- chiedere l'altezza;
- calcolare l'area;
- stampare il risultato.

## Formula

```text
area = base * altezza
```

## Esempio di esecuzione

```text
Inserisci la base: 5
Inserisci l'altezza: 3
Area del rettangolo: 15.0
```

## Vincolo aggiuntivo

Base e altezza devono essere maggiori di zero.

Se l'utente inserisce un numero minore o uguale a zero, il programma deve richiederlo.

Qui non basta controllare solo la conversione: bisogna controllare anche il valore.

Sì, l'utente può sbagliare anche quando finalmente scrive un numero. La creatività umana non ha limiti.

# Esercizio 6 - Mini convertitore chilometri/miglia

## Obiettivo

Scrivere un programma che converta una distanza da chilometri a miglia oppure da miglia a chilometri.

## File da creare

```text
Esercizio06_ConvertitoreDistanze.java
```

## Requisiti

Il programma deve chiedere:

- valore della distanza;
- tipo di conversione.

Tipi di conversione ammessi:

```text
K
M
```

Dove:

- `K` significa da chilometri a miglia;
- `M` significa da miglia a chilometri.

## Formule

Da chilometri a miglia:

```text
miglia = chilometri * 0.621371
```

Da miglia a chilometri:

```text
chilometri = miglia / 0.621371
```

## Metodi richiesti

```java
public static double leggiDoublePositivo(Scanner scanner, String messaggio)
```

```java
public static char leggiTipoConversione(Scanner scanner, String messaggio)
```

```java
public static double converti(double valore, char tipo)
```

```java
public static void stampaRisultato(double valore, char tipo, double risultato)
```

## Esempio

```text
Inserisci la distanza: 10
Conversione K=km->miglia, M=miglia->km: K
10.0 km corrispondono a 6.21371 miglia
```


## Requisiti

Il programma deve:

- leggere la scelta con un metodo robusto;
- accettare solo valori da `0` a `4`;
- chiedere due numeri solo se la scelta è diversa da `0`;
- eseguire l'operazione scelta;
- gestire la divisione per zero;
- ripetere il menu finché l'utente non sceglie `0`.

## Metodi richiesti

```java

```

```java

```

```java
public static void stampaRisultato(double risultato)
```

```java
public static void stampaMenu()
```

## Suggerimento

Puoi usare un ciclo:

```java
while (scelta != 0) {
    ...
}
```

oppure:

```java
do {
    ...
} while (scelta != 0);
```

Usa la forma che ritieni più chiara.

# Sintesi

Questi esercizi consolidano una regola fondamentale:

> un programma robusto non si limita a funzionare quando l'utente inserisce dati corretti, ma gestisce anche gli input sbagliati in modo controllato.

In Java questo richiede:

- lettura ordinata dell'input;
- conversione esplicita;
- gestione degli errori;
- metodi riusabili;
- responsabilità ben separate.

Quando il codice è organizzato bene, il `main` diventa più leggibile e i dettagli ripetitivi vengono spostati in metodi dedicati.

Non è magia. È solo disciplina. Terribilmente meno scenografica, ma molto più utile.
