# 01. Introduzione a `try`, `catch` e gestione degli errori di conversione

## Obiettivo

In questa attività imparerai a gestire in modo controllato alcuni errori che possono verificarsi durante l'esecuzione di un programma Java.

L'obiettivo non è ancora studiare tutte le eccezioni di Java. Sarebbe un modo elegante per rendere antipatica la programmazione prima del necessario.

L'obiettivo è più concreto:

- capire che alcuni errori avvengono mentre il programma è in esecuzione;
- capire perché una conversione da `String` a numero può fallire;
- usare `try` e `catch` per evitare che il programma si interrompa bruscamente;
- applicare questa tecnica alla lettura dell'input da tastiera;
- prepararsi al laboratorio sugli input robusti e sulle utility statiche.

---

## 1. Errori di compilazione ed errori a runtime

In Java possiamo incontrare errori di tipo diverso.

Per ora ci interessano soprattutto due categorie:

| Tipo di errore | Quando avviene | Esempio |
|---|---|---|
| Errore di compilazione | Prima dell'esecuzione | manca un `;`, nome di variabile sbagliato, tipo non compatibile |
| Errore a runtime | Durante l'esecuzione | conversione non valida, divisione per zero in alcuni casi, accesso non valido a dati |

---

## 2. Esempio di errore di compilazione

Osserva questo codice:

```java
public class Main {
    public static void main(String[] args) {
        int numero = "ciao";
        System.out.println(numero);
    }
}
```

Questo programma non compila.

Il problema è qui:

```java
int numero = "ciao";
```

Stiamo cercando di assegnare una stringa a una variabile di tipo `int`.

Java se ne accorge prima dell'esecuzione.

Il compilatore segnala un errore e il programma non parte.

---

## 3. Esempio di errore a runtime

Osserva invece questo codice:

```java
public class Main {
    public static void main(String[] args) {
        String testo = "ciao";
        int numero = Integer.parseInt(testo);

        System.out.println(numero);
    }
}
```

Questo programma può compilare.

La sintassi è corretta.

Il problema nasce durante l'esecuzione, quando Java prova a convertire:

```java
"ciao"
```

in un numero intero.

Questa conversione non è possibile.

Il programma termina con un errore.

---

## 4. Conversione da `String` a `int`

Per convertire una stringa in un numero intero si usa:

```java
Integer.parseInt(...)
```

Esempio corretto:

```java
public class Main {
    public static void main(String[] args) {
        String testo = "25";
        int numero = Integer.parseInt(testo);

        System.out.println("Numero: " + numero);
    }
}
```

In questo caso la stringa contiene un valore intero valido.

La conversione riesce.

Output previsto:

```text
Numero: 25
```

---

## 5. Conversione non valida

Ora prova questo esempio:

```java
public class Main {
    public static void main(String[] args) {
        String testo = "venti";
        int numero = Integer.parseInt(testo);

        System.out.println("Numero: " + numero);
    }
}
```

La stringa `"venti"` non contiene cifre numeriche.

Java non può convertirla in `int`.

Il programma genera una eccezione chiamata:

```text
NumberFormatException
```

---

## 6. Che cos'è una eccezione

Una eccezione è un evento anomalo che si verifica durante l'esecuzione del programma.

Nel nostro caso, l'anomalia è questa:

> il programma ha chiesto a Java di convertire una stringa in numero, ma la stringa non contiene un numero valido.

Quando l'eccezione non viene gestita, il programma si interrompe.

Esempio:

```java
String testo = "abc";
int numero = Integer.parseInt(testo);
```

Qui Java prova a convertire `"abc"` in un intero.

Non può farlo.

Quindi lancia una `NumberFormatException`.

---

## 7. Perché serve `try/catch`

Quando una parte del programma può generare un errore a runtime, possiamo racchiuderla in una struttura `try/catch`.

La forma generale è:

```java
try {
    // codice che potrebbe generare un errore
} catch (TipoEccezione nomeVariabile) {
    // codice da eseguire se si verifica quell'errore
}
```

Significato:

- nel blocco `try` inseriamo il codice che vogliamo provare a eseguire;
- nel blocco `catch` inseriamo il codice da eseguire se si verifica uno specifico errore;
- se non si verifica nessun errore, il blocco `catch` viene ignorato;
- se si verifica l'errore indicato nel `catch`, il programma non termina bruscamente.

---

## 8. Primo esempio con `try/catch`

Crea un file chiamato:

```text
EsempioTryCatch.java
```

Inserisci il seguente codice:

```java
public class EsempioTryCatch {
    public static void main(String[] args) {
        String testo = "abc";

        try {
            int numero = Integer.parseInt(testo);
            System.out.println("Numero convertito: " + numero);
        } catch (NumberFormatException e) {
            System.out.println("Errore: la stringa non contiene un intero valido.");
        }

        System.out.println("Il programma continua.");
    }
}
```

Compila:

```bash
javac EsempioTryCatch.java
```

Esegui:

```bash
java EsempioTryCatch
```

Output previsto:

```text
Errore: la stringa non contiene un intero valido.
Il programma continua.
```

---

## 9. Analisi dell'esempio

La parte potenzialmente pericolosa è questa:

```java
int numero = Integer.parseInt(testo);
```

Per questo viene inserita nel blocco `try`.

Se la conversione riesce, viene eseguita questa istruzione:

```java
System.out.println("Numero convertito: " + numero);
```

Se la conversione fallisce, viene eseguito il blocco `catch`:

```java
catch (NumberFormatException e) {
    System.out.println("Errore: la stringa non contiene un intero valido.");
}
```

Il programma non si interrompe.

Infatti dopo il `catch` viene comunque eseguita questa istruzione:

```java
System.out.println("Il programma continua.");
```

---

## 10. Provare un caso valido

Modifica il valore della variabile:

```java
String testo = "123";
```

Il programma completo diventa:

```java
public class EsempioTryCatch {
    public static void main(String[] args) {

        try {
            int numero = Integer.parseInt(testo);
            System.out.println("Numero convertito: " + numero);
        } catch (NumberFormatException e) {
            System.out.println("Errore: la stringa non contiene un intero valido.");
        }

        System.out.println("Il programma continua.");
    }
}
```

Output previsto:

```text
Numero convertito: 123
Il programma continua.
```

In questo caso il blocco `catch` non viene eseguito.

## 11. Lettura da tastiera e conversione

Ora applichiamo lo stesso concetto all'input da tastiera.

Crea un file chiamato:

```text
EsempioInputProtetto.java
```

Inserisci il seguente codice:

```java
import java.util.Scanner;

public class EsempioInputProtetto {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Inserisci un numero intero: ");
        String testo = scanner.nextLine();

        try {
            int numero = Integer.parseInt(testo);
            System.out.println("Numero valido: " + numero);
        } catch (NumberFormatException e) {
            System.out.println("Errore: devi inserire un numero intero valido.");
        }

        scanner.close();
    }
}
```

Compila:

```bash
javac EsempioInputProtetto.java
```

Esegui:

```bash
java EsempioInputProtetto
```

Prova prima con:

```text
42
```

Poi esegui di nuovo il programma e prova con:

```text
ciao
```

---

## 12. Perché leggiamo con `nextLine()`

In questo esempio usiamo:

```java
String testo = scanner.nextLine();
```

Questa istruzione legge sempre una riga di testo.

Poi convertiamo il testo con:

```java
int numero = Integer.parseInt(testo);
```

Questo approccio separa due operazioni diverse:

| Operazione | Istruzione |
|---|---|
| leggere ciò che l'utente ha scritto | `scanner.nextLine()` |
| convertire il testo in numero | `Integer.parseInt(testo)` |

Questa separazione rende il programma più controllabile.

Invece, usando direttamente `nextInt()`, lettura e conversione avvengono insieme.

Questo è più breve, ma meno comodo quando vogliamo gestire bene gli errori.

## 13. Conversione da `String` a `double`

Per convertire una stringa in un numero reale si usa:

```java
Double.parseDouble(...)
```

Esempio:

```java
public class EsempioParseDouble {
    public static void main(String[] args) {
        String testo = "12.5";
        double valore = Double.parseDouble(testo);

        System.out.println("Valore: " + valore);
    }
}
```

Output previsto:

```text
Valore: 12.5
```

Attenzione: in questa esercitazione useremo il punto come separatore decimale:

```text
12.5
```

Non useremo la virgola:

```text
12,5
```

La gestione della virgola italiana sarà trattata eventualmente come variante successiva.

## 14. Esempio protetto con `double`

Crea un file chiamato:

```text
EsempioDoubleProtetto.java
```

Inserisci il codice:

```java
import java.util.Scanner;

public class EsempioDoubleProtetto {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Inserisci un numero reale: ");
        String testo = scanner.nextLine();

        try {
            double valore = Double.parseDouble(testo);
            System.out.println("Valore valido: " + valore);
        } catch (NumberFormatException e) {
            System.out.println("Errore: devi inserire un numero reale valido.");
        }

        scanner.close();
    }
}
```

Prova con:

```text
15.75
```

Poi prova con:

```text
quindici
```

Nel primo caso la conversione riesce.

Nel secondo caso il programma intercetta l'errore e mostra un messaggio controllato.

## 15. Rendere la richiesta ripetibile

Finora, se l'utente sbaglia, il programma mostra un errore e termina.

Possiamo migliorare il comportamento usando un ciclo.

Crea un file chiamato:

```text
EsempioRipetiFincheValido.java
```

Inserisci il seguente codice:

```java
import java.util.Scanner;

public class EsempioRipetiFincheValido {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int numero;

        while (true) {
            System.out.print("Inserisci un numero intero: ");
            String testo = scanner.nextLine();

            try {
                numero = Integer.parseInt(testo);
                break;
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero intero valido.");
            }
        }

        System.out.println("Numero acquisito: " + numero);

        scanner.close();
    }
}
```

---

## 16. Analisi del ciclo `while (true)`

Il ciclo:

```java
while (true) {
    ...
}
```

crea una ripetizione potenzialmente infinita.

In questo caso non è un errore, perché il ciclo viene interrotto da:

```java
break;
```

Il `break` viene eseguito solo quando la conversione riesce.

Quindi il significato è:

> continua a chiedere il valore finché l'utente non inserisce un numero valido.

Sì, è una piccola trattativa con l'utente. Il programma insiste, l'utente prima o poi collabora. In teoria.

---

## 17. Da codice ripetuto a metodo statico

Se dobbiamo leggere numeri interi in più punti del programma, non conviene copiare sempre lo stesso codice.

Possiamo creare un metodo statico.

Crea un file chiamato:

```text
EsempioMetodoLeggiIntero.java
```

Inserisci il codice:

```java
import java.util.Scanner;

public class EsempioMetodoLeggiIntero {

    public static int leggiIntero(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Integer.parseInt(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero intero valido.");
            }
        }
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int eta = leggiIntero(scanner, "Inserisci la tua età: ");

        System.out.println("Età inserita: " + eta);

        scanner.close();
    }
}
```

---

## 18. Perché il metodo usa `return`

Dentro il metodo:

```java
return Integer.parseInt(testo);
```

succedono due cose:

1. Java prova a convertire la stringa in intero;
2. se la conversione riesce, il valore viene restituito al chiamante.

Quando viene eseguito `return`, il metodo termina immediatamente.

Se invece la conversione fallisce, il `return` non viene completato e si passa al blocco `catch`.

---

## 19. Metodo per leggere un `double`

La stessa idea può essere applicata ai numeri reali.

```java
public static double leggiDouble(Scanner scanner, String messaggio) {
    while (true) {
        System.out.print(messaggio);
        String testo = scanner.nextLine();

        try {
            return Double.parseDouble(testo);
        } catch (NumberFormatException e) {
            System.out.println("Errore: devi inserire un numero reale valido.");
        }
    }
}
```

Anche qui:

- il metodo chiede un valore;
- legge una riga;
- prova a convertirla;
- se la conversione fallisce, mostra un errore e ripete;
- se la conversione riesce, restituisce il valore.

## 21. Errori comuni

### Errore 1: pensare che `try/catch` corregga automaticamente il dato

Questo codice non trasforma `"ciao"` in un numero.

```java
try {
    int numero = Integer.parseInt("ciao");
} catch (NumberFormatException e) {
    System.out.println("Errore");
}
```

Il `catch` non corregge il valore.

Il `catch` permette solo di gestire l'errore in modo controllato.

---

### Errore 2: mettere nel `try` troppo codice inutile

Meglio inserire nel `try` solo il codice che può davvero generare l'eccezione che vogliamo gestire.

Esempio chiaro:

```java
try {
    int numero = Integer.parseInt(testo);
    System.out.println("Numero: " + numero);
} catch (NumberFormatException e) {
    System.out.println("Errore: numero non valido.");
}
```

---

### Errore 3: usare `catch` troppo generici

In questa fase useremo:

```java
catch (NumberFormatException e)
```

Non useremo:

```java
catch (Exception e)
```

`Exception` è molto generico.

Per ora è meglio intercettare l'errore specifico che ci interessa.

---

### Errore 4: dimenticare che `parseDouble` usa il punto

In questa esercitazione:

```text
12.5
```

è valido.

```text
12,5
```

non è valido.

## 23. Sintesi

Quando leggiamo input da tastiera, l'utente può inserire valori non validi.

Se chiediamo un numero, l'utente può scrivere:

```text
ciao
```

oppure:

```text
venti
```

oppure un valore scritto in un formato non previsto.

Per evitare che il programma si interrompa, possiamo:

1. leggere sempre una riga di testo con `nextLine()`;
2. provare a convertire il testo con `Integer.parseInt(...)` o `Double.parseDouble(...)`;
3. racchiudere la conversione in un blocco `try`;
4. gestire l'errore con un blocco `catch`;
5. eventualmente ripetere la richiesta finché il valore non è valido.

Questo schema sarà usato nel laboratorio successivo per costruire utility statiche di input più ordinate e riusabili.
