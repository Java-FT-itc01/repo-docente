# 02. Scanner, `nextLine()` e input misto

## Obiettivo

In questa attività analizzerai uno dei problemi più frequenti quando si usa `Scanner` nei programmi Java da console:

> usare insieme metodi come `nextInt()`, `nextDouble()` e `nextLine()` può produrre comportamenti inattesi.

Il problema più comune è questo:

- il programma chiede un numero;
- poi chiede una stringa;
- la stringa sembra essere saltata;
- il partecipante guarda il codice;
- il codice sembra giusto;
- Java, naturalmente, non si scusa.

L'obiettivo è capire **perché succede** e imparare una strategia più ordinata per leggere input da tastiera.

Prima di iniziare dovresti conoscere:

- dichiarazione di variabili;

- uso di `System.out.print()` e `System.out.println()`;
- concetto base di metodo;
- concetto base di errore a runtime;

## 2. Creazione di uno Scanner

Per leggere input da tastiera usiamo:

```java
import java.util.Scanner;
```

e poi:

```java
Scanner scanner = new Scanner(System.in);
```

`System.in` rappresenta normalmente l'input da tastiera.

Esempio minimo:

```java
import java.util.Scanner;

public class EsempioScannerBase {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Inserisci il tuo nome: ");
        String nome = scanner.nextLine();

        System.out.println("Ciao " + nome);

        scanner.close();
    }
}
```

---

## 3. Due famiglie di metodi di lettura

Con `Scanner` possiamo usare metodi diversi.

Alcuni metodi leggono un valore già convertito:

```java
nextInt()
nextDouble()
nextBoolean()
```

Altri leggono testo:

```java
next()
nextLine()
```

Per ora ci interessano soprattutto questi:

| Metodo | Cosa legge | Tipo restituito |
|---|---|---|
| `nextInt()` | un intero | `int` |
| `nextDouble()` | un numero reale | `double` |
| `next()` | una parola fino allo spazio | `String` |
| `nextLine()` | tutta la riga fino a Invio | `String` |

---

## 6. Il problema: `nextInt()` seguito da `nextLine()`

Ora osserva un caso molto comune.

Crea un file chiamato:

```text
ProblemaNextIntNextLine.java
```

Inserisci il codice:

```java
import java.util.Scanner;

public class ProblemaNextIntNextLine {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Inserisci la tua età: ");
        int eta = scanner.nextInt();

        System.out.print("Inserisci il tuo nome completo: ");
        String nome = scanner.nextLine();

        System.out.println("Età: " + eta);
        System.out.println("Nome: " + nome);

        scanner.close();
    }
}
```

Compila:

```bash
javac ProblemaNextIntNextLine.java
```

Esegui:

```bash
java ProblemaNextIntNextLine
```

Prova a inserire:

```text
25
```

Osserva cosa succede.

Probabilmente il programma non ti permette di scrivere davvero il nome.

La variabile `nome` risulta vuota.

---

## 7. Perché succede

Quando scrivi:

```text
25
```

e premi Invio, nella console entrano due cose:

```text
25
[Invio]
```

Il metodo:

```java
scanner.nextInt();
```

legge solo il numero:

```text
25
```

ma non consuma completamente il carattere di fine riga prodotto dal tasto Invio.

Poi arriva:

```java
scanner.nextLine();
```

`nextLine()` legge ciò che resta fino alla fine della riga.

Ma la fine della riga è già lì, rimasta dopo il numero.

Quindi `nextLine()` legge una stringa vuota.

Il programma non è impazzito. Sta solo facendo esattamente quello che gli abbiamo chiesto.

---

## 8. Il concetto di buffer

Possiamo immaginare che l'input digitato dall'utente venga messo in una specie di area temporanea chiamata **buffer**.

Quando digiti:

```text
25
```

e premi Invio, il buffer contiene qualcosa di simile:

```text
25\n
```

dove `\n` rappresenta la fine della riga.

`nextInt()` consuma il valore numerico:

```text
25
```

ma lascia nel buffer:

```text
\n
```

Poi `nextLine()` trova subito `\n` e considera terminata la riga.

Risultato:

```java
String nome = "";
```

cioè una stringa vuota.

---

## 9. Prima soluzione: consumare il fine riga

Una possibile soluzione è aggiungere un `nextLine()` subito dopo `nextInt()`.

Crea un file chiamato:

```text
SoluzioneConsumaInvio.java
```

Inserisci il codice:

```java
import java.util.Scanner;

public class SoluzioneConsumaInvio {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Inserisci la tua età: ");
        int eta = scanner.nextInt();
        scanner.nextLine();

        System.out.print("Inserisci il tuo nome completo: ");
        String nome = scanner.nextLine();

        System.out.println("Età: " + eta);
        System.out.println("Nome: " + nome);

        scanner.close();
    }
}
```

La riga:

```java
scanner.nextLine();
```

subito dopo `nextInt()` serve a consumare il fine riga rimasto nel buffer.

Questa soluzione funziona, ma richiede attenzione.

Se dimentichi quella riga, il problema torna.

E naturalmente tornerà nel momento meno comodo, perché il software ha senso dell'umorismo.

Il problema non riguarda solo `nextInt()`.

Può comparire anche con `nextDouble()`.


Prova a eseguirlo.

Anche qui la descrizione può risultare vuota.

---

## 11. Seconda soluzione: leggere tutto con `nextLine()`

Una soluzione più ordinata consiste nel leggere sempre righe di testo con:

```java
scanner.nextLine()
```

e poi convertire i valori quando serve.

Invece di scrivere:

```java
int eta = scanner.nextInt();
```

possiamo scrivere:

```java
String testoEta = scanner.nextLine();
int eta = Integer.parseInt(testoEta);
```

In questo modo separiamo due operazioni:

| Operazione | Codice |
|---|---|
| lettura della riga | `scanner.nextLine()` |
| conversione in intero | `Integer.parseInt(testoEta)` |

## 13. Aggiunta di `try/catch`

Per gestire un input non numerico possiamo usare `try/catch`.

Crea un file chiamato:

```text
SoluzioneNextLineTryCatch.java
```

Inserisci il codice:

```java
import java.util.Scanner;

public class SoluzioneNextLineTryCatch {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Inserisci la tua età: ");
        String testoEta = scanner.nextLine();

        try {
            int eta = Integer.parseInt(testoEta);

            System.out.print("Inserisci il tuo nome completo: ");
            String nome = scanner.nextLine();

            System.out.println("Età: " + eta);
            System.out.println("Nome: " + nome);
        } catch (NumberFormatException e) {
            System.out.println("Errore: devi inserire un'età numerica valida.");
        }

        scanner.close();
    }
}
```

Questo programma non si interrompe bruscamente se l'età non è valida.

Mostra invece un messaggio controllato.

## 14. Ripetere la richiesta finché il valore è valido

Un programma più comodo non dovrebbe terminare subito al primo errore.

Può chiedere di nuovo il valore.

Crea un file chiamato:

```text
InputRipetutoEta.java
```

Inserisci il codice:

```java
import java.util.Scanner;

public class InputRipetutoEta {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int eta;

        while (true) {
            System.out.print("Inserisci la tua età: ");
            String testoEta = scanner.nextLine();

            try {
                eta = Integer.parseInt(testoEta);
                break;
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero intero.");
            }
        }

        System.out.print("Inserisci il tuo nome completo: ");
        String nome = scanner.nextLine();

        System.out.println("Età: " + eta);
        System.out.println("Nome: " + nome);

        scanner.close();
    }
}
```

Prova prima a inserire:

```text
abc
```

Poi inserisci:

```text
25
```

Il programma continua a chiedere l'età finché non riceve un intero valido.

## 15. Creare una utility statica

Se la lettura robusta serve più volte, è meglio trasformarla in un metodo statico.

Crea un file chiamato:

```text
UtilityInputIntero.java
```

Inserisci il codice:

```java
import java.util.Scanner;

public class UtilityInputIntero {

    public static int leggiIntero(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Integer.parseInt(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero intero valido.");
            }
        }
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int eta = leggiIntero(scanner, "Inserisci la tua età: ");
        int anno = leggiIntero(scanner, "Inserisci l'anno corrente: ");

        System.out.println("Età: " + eta);
        System.out.println("Anno corrente: " + anno);

        scanner.close();
    }
}
```

Questo codice è più ordinato perché la logica di lettura robusta si trova in un solo punto.

## 16. Utility per `double`

```java
import java.util.Scanner;
```

```text
12.5
```

Non usare la virgola:

```text
12,5
```

## 17. Quando usare `nextInt()` e quando usare `nextLine()`

`nextInt()` e `nextDouble()` sono comodi per esempi molto semplici.

Tuttavia, quando il programma deve leggere input misti, come numeri e stringhe, spesso conviene usare sempre:

```java
nextLine()
```

e poi convertire.

Schema consigliato:

```java
String testo = scanner.nextLine();
int valore = Integer.parseInt(testo);
```

oppure:

```java
String testo = scanner.nextLine();
double valore = Double.parseDouble(testo);
```

Con `try/catch`:

```java
try {
    int valore = Integer.parseInt(testo);
} catch (NumberFormatException e) {
    System.out.println("Errore: valore non valido.");
}
```

---

## 18. Regola pratica

Nei programmi didattici più ordinati:

- crea un solo `Scanner`;
- usa preferibilmente `nextLine()` per leggere l'input;
- converti con `Integer.parseInt()` o `Double.parseDouble()`;
- proteggi la conversione con `try/catch`;
- se l'input è sbagliato, mostra un messaggio e chiedi di nuovo il valore;
- non mescolare senza attenzione `nextInt()` / `nextDouble()` con `nextLine()`.

## 19. Esercizio guidato finale

Crea un file chiamato:

```text
RegistrazioneProdotto.java
```

Il programma deve chiedere:

- nome prodotto;
- prezzo;
- quantità;
- categoria.

Regole:

- nome prodotto e categoria devono essere letti come stringhe;
- prezzo deve essere letto come `double`;
- quantità deve essere letta come `int`;
- prezzo e quantità devono essere acquisiti con metodi statici robusti;
- il programma deve continuare a chiedere prezzo e quantità finché l'input non è valido.

Codice di partenza:

```java
import java.util.Scanner;

public class RegistrazioneProdotto {

    public static int leggiIntero(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Integer.parseInt(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero intero valido.");
            }
        }
    }

    public static double leggiDouble(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Double.parseDouble(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero reale valido.");
            }
        }
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Nome prodotto: ");
        String nome = scanner.nextLine();

        double prezzo = leggiDouble(scanner, "Prezzo: ");
        int quantita = leggiIntero(scanner, "Quantità: ");

        System.out.print("Categoria: ");
        String categoria = scanner.nextLine();

        double totale = prezzo * quantita;

        System.out.println();
        System.out.println("Scheda prodotto");
        System.out.println("Nome: " + nome);
        System.out.println("Prezzo: " + prezzo);
        System.out.println("Quantità: " + quantita);
        System.out.println("Categoria: " + categoria);
        System.out.println("Totale: " + totale);

        scanner.close();
    }
}
```

## 21. Sintesi finale

Quando si usa `Scanner`, bisogna distinguere tra:

- lettura diretta di valori già convertiti;
- lettura di righe testuali;
- conversione manuale del testo;
- gestione degli errori di conversione.

Il problema più frequente nasce quando si mescolano:

```java
nextInt()
```

o:

```java
nextDouble()
```

con:

```java
nextLine()
```

Per evitare confusione, una strategia molto efficace è:

1. leggere sempre con `nextLine()`;
2. convertire con `Integer.parseInt()` o `Double.parseDouble()`;
3. proteggere la conversione con `try/catch`;
4. incapsulare la logica ripetuta in metodi statici.

Questo schema sarà usato nel laboratorio successivo per costruire programmi più robusti, leggibili e riusabili.
