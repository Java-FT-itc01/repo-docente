# 03. LAB06 - Utility statiche e input robusto

## Obiettivo del laboratorio

In questo laboratorio costruirai una piccola calcolatrice da console usando metodi statici e input robusto.

L'obiettivo non è soltanto ottenere un risultato numerico.

L'obiettivo è imparare a scrivere un programma più ordinato, separando:

- lettura dell'input;
- conversione dei dati;
- controllo degli errori;
- logica di calcolo;
- stampa del risultato.

Nei laboratori precedenti hai visto che leggere direttamente con `nextInt()` o `nextDouble()` può essere comodo, ma può creare problemi quando l'utente inserisce dati non validi o quando si mescolano numeri e stringhe.

In questo laboratorio userai una strategia più controllata:

1. leggere sempre una riga di testo con `nextLine()`;
2. convertire il testo con `Integer.parseInt()` o `Double.parseDouble()`;
3. gestire eventuali errori con `try/catch`;
4. racchiudere il codice ripetuto in metodi statici.

## 6. Protezione della conversione con `try/catch`

Modifica il `main` così:

```java
public static void main(String[] args) {
    Scanner scanner = new Scanner(System.in);

    System.out.println("Calcolatrice robusta");

    System.out.print("Inserisci il primo numero: ");
```

```text
ciao
```

Nel secondo caso il programma non termina con un errore tecnico, ma mostra un messaggio controllato.

Questo è il punto: l'utente sbaglia, il programma non sviene.

## 7. Ripetere la richiesta finché il numero è valido

Un programma migliore non dovrebbe terminare subito al primo errore.

Dovrebbe chiedere di nuovo il valore.

Modifica il `main` così:

```java
public static void main(String[] args) {
    Scanner scanner = new Scanner(System.in);

    System.out.println("Calcolatrice robusta");

    double numero;

    while (true) {
        System.out.print("Inserisci il primo numero: ");
        String testo = scanner.nextLine();

        try {
            numero = Double.parseDouble(testo);
            break;
        } catch (NumberFormatException e) {
            System.out.println("Errore: devi inserire un numero reale valido.");
        }
    }

    System.out.println("Numero acquisito: " + numero);

    scanner.close();
}
```

Compila ed esegui.

Prova a inserire valori non validi, poi un valore valido.

Esempio:

```text
abc
dodici
12.5
```

Il programma continua a chiedere il valore finché l'input non è corretto.

## 8. Trasformare la lettura in un metodo statico

Il codice di lettura robusta è utile, ma nel `main` occupa troppo spazio.

Creiamo un metodo statico.

Aggiungi questo metodo dentro la classe, ma fuori dal `main`:

```java
public static double leggiDouble(Scanner scanner, String messaggio) {
    while (true) {
        System.out.print(messaggio);
        String testo = scanner.nextLine();

        try {
            return Double.parseDouble(testo);
        } catch (NumberFormatException e) {
            System.out.println("Errore: devi inserire un numero reale valido.");
        }
    }
}
```

La classe avrà questa forma:

```java
import java.util.Scanner;

public class CalcolatriceRobusta {

    public static double leggiDouble(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Double.parseDouble(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero reale valido.");
            }
        }
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("Calcolatrice robusta");

        double numero = leggiDouble(scanner, "Inserisci il primo numero: ");

        System.out.println("Numero acquisito: " + numero);

        scanner.close();
    }
}
```

Compila ed esegui.

---

## 9. Analisi del metodo `leggiDouble`

Il metodo:

```java
public static double leggiDouble(Scanner scanner, String messaggio)
```

ha questi elementi:

| Elemento | Significato |
|---|---|
| `public` | il metodo è accessibile dall'esterno della classe |
| `static` | il metodo può essere chiamato dal `main` senza creare oggetti |
| `double` | il metodo restituisce un valore di tipo `double` |
| `leggiDouble` | nome del metodo |
| `Scanner scanner` | oggetto usato per leggere da tastiera |
| `String messaggio` | testo da mostrare all'utente |

Dentro il metodo c'è un ciclo:

```java
while (true)
```

Il ciclo continua finché non viene eseguito un `return`.

Quando questa riga riesce:

```java
return Double.parseDouble(testo);
```

il metodo restituisce il numero convertito e termina.

Se la conversione fallisce, viene eseguito il `catch` e il ciclo ricomincia.

---

## 10. Perché passiamo lo `Scanner` come parametro

Il metodo riceve lo `Scanner`:

```java
leggiDouble(scanner, "Inserisci il primo numero: ");
```

In questo modo non creiamo un nuovo `Scanner` dentro ogni metodo.

Usiamo sempre lo stesso oggetto creato nel `main`.

Questa scelta rende il programma più ordinato e riduce problemi legati alla gestione dell'input.

Regola pratica:

> in un programma console semplice, crea uno Scanner nel `main` e passalo ai metodi che devono leggere input.

## 12. Lettura dell'operatore

La calcolatrice deve chiedere anche un operatore.

Gli operatori ammessi sono:

```text
+ - * /
```

Aggiungi questo metodo alla classe:

```java
public static char leggiOperatore(Scanner scanner, String messaggio) {
    while (true) {
        System.out.print(messaggio);
        String testo = scanner.nextLine();

        if (testo.length() == 1) {
            char operatore = testo.charAt(0);

            if (operatore == '+' || operatore == '-' || operatore == '*' || operatore == '/') {
                return operatore;
            }
        }

        System.out.println("Operatore non valido. Inserisci uno tra +, -, * oppure /.");
    }
}
```

---

## 13. Analisi di `leggiOperatore`

Il metodo legge sempre una riga:

```java
String testo = scanner.nextLine();
```

Poi controlla che la riga abbia un solo carattere:

```java
if (testo.length() == 1)
```

Se la lunghezza è 1, prende il primo carattere:

```java
char operatore = testo.charAt(0);
```

Poi controlla che il carattere sia uno degli operatori ammessi:

```java
if (operatore == '+' || operatore == '-' || operatore == '*' || operatore == '/')
```

Se l'operatore è valido, viene restituito con:

```java
return operatore;
```

Se invece l'input non è valido, il metodo mostra un messaggio e ripete la richiesta.

---

## 14. Micro-esempio su `charAt(0)`

Per capire meglio `charAt(0)`, osserva questo esempio:

```java
String testo = "+";
char simbolo = testo.charAt(0);

System.out.println(simbolo);
```

La stringa contiene un solo carattere.

`charAt(0)` prende il carattere in posizione zero.

In Java gli indici partono da zero.

Quindi, nella stringa:

```text
+
```

l'unico carattere disponibile è in posizione `0`.

## 16. Metodo `calcola`

Ora aggiungiamo la logica della calcolatrice.

Aggiungi questo metodo alla classe:

```java
public static double calcola(double a, double b, char operatore) {
    switch (operatore) {
        case '+':
            return a + b;
        case '-':
            return a - b;
        case '*':
            return a * b;
        case '/':
            if (b == 0) {
                return Double.NaN;
            }
            return a / b;
        default:
            return Double.NaN;
    }
}
```

---

## 17. Analisi del metodo `calcola`

Il metodo riceve:

```java
double a
double b
char operatore
```

e restituisce:

```java
double
```

Usa uno `switch` perché l'operatore può assumere pochi valori distinti.

Nel caso della divisione:

```java
case '/':
    if (b == 0) {
        return Double.NaN;
    }
    return a / b;
```

controlliamo se il secondo numero è zero.

La divisione per zero non viene calcolata.

Il metodo restituisce:

```java
Double.NaN
```

## 18. Che cos'è `Double.NaN`

`Double.NaN` significa:

```text
Not a Number
```

Rappresenta un risultato numerico non valido.

In questo laboratorio lo usiamo come segnale per dire:

> il risultato non può essere calcolato correttamente.

Esempi:

- divisione per zero;
- operatore non previsto.

Per controllare se un risultato è `NaN`, si usa:

```java
Double.isNaN(risultato)
```

## 19. Metodo `stampaRisultato`

Aggiungi questo metodo:

```java
public static void stampaRisultato(double a, double b, char operatore, double risultato) {
    if (Double.isNaN(risultato)) {
        System.out.println("Impossibile mostrare un risultato valido.");
    } else {
        System.out.println("Risultato: " + a + " " + operatore + " " + b + " = " + risultato);
    }
}
```

Questo metodo si occupa solo della stampa finale.

Se il risultato non è valido, mostra un messaggio di errore.

Se il risultato è valido, mostra l'operazione completa.

I dettagli sono spostati nei metodi.

Questa è una buona organizzazione del codice.

## 21. Codice finale completo

Confronta il tuo file con questa versione completa:

```java
import java.util.Scanner;

public class CalcolatriceRobusta {

    public static double leggiDouble(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Double.parseDouble(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un numero reale valido.");
            }
        }
    }

    public static char leggiOperatore(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            if (testo.length() == 1) {
                char operatore = testo.charAt(0);

                if (operatore == '+' || operatore == '-' || operatore == '*' || operatore == '/') {
                    return operatore;
                }
            }

            System.out.println("Operatore non valido. Inserisci uno tra +, -, * oppure /.");
        }
    }

    public static double calcola(double a, double b, char operatore) {
        switch (operatore) {
            case '+':
                return a + b;
            case '-':
                return a - b;
            case '*':
                return a * b;
            case '/':
                if (b == 0) {
                    return Double.NaN;
                }
                return a / b;
            default:
                return Double.NaN;
        }
    }

    public static void stampaRisultato(double a, double b, char operatore, double risultato) {
        if (Double.isNaN(risultato)) {
            System.out.println("Impossibile mostrare un risultato valido.");
        } else {
            System.out.println("Risultato: " + a + " " + operatore + " " + b + " = " + risultato);
        }
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("Calcolatrice robusta");

        double primoNumero = leggiDouble(scanner, "Inserisci il primo numero: ");
        double secondoNumero = leggiDouble(scanner, "Inserisci il secondo numero: ");
        char operatore = leggiOperatore(scanner, "Inserisci l'operatore (+, -, *, /): ");

        double risultato = calcola(primoNumero, secondoNumero, operatore);

        stampaRisultato(primoNumero, secondoNumero, operatore, risultato);

        scanner.close();
    }
}
```

## 23. Nota su `scanner.close()`

Nel codice finale chiudiamo lo Scanner alla fine:

```java
scanner.close();
```

In questo programma va bene perché dopo quella riga il programma termina.

Lo Scanner è collegato a:

```java
System.in
```

cioè l'input da tastiera.

Chiudere lo Scanner significa chiudere anche `System.in`.

Per questo motivo non bisogna chiudere lo Scanner a metà programma se si dovrà ancora leggere da tastiera.

## 24. Regola sui numeri decimali

In questo laboratorio usa il punto come separatore decimale:

```text
12.5
```

Non usare la virgola:

```text
12,5
```

Il motivo è che:

```java
Double.parseDouble("12.5")
```

funziona, mentre:

```java
Double.parseDouble("12,5")
```

genera una `NumberFormatException`.

La gestione della virgola italiana può essere aggiunta come variante successiva.

## 27. Sintesi finale

In questo laboratorio hai costruito una calcolatrice più robusta usando metodi statici.

Il programma finale è migliore di una versione scritta tutta nel `main` perché:

- il `main` è più leggibile;
- la lettura dei numeri è riusabile;
- gli errori di input non bloccano il programma;
- l'operatore viene validato;
- la logica di calcolo è separata dalla logica di input;
- la stampa del risultato è gestita in un metodo dedicato.

Lo schema usato in questo laboratorio è importante:

```text
leggere -> convertire -> validare -> elaborare -> stampare
```

Questo schema tornerà spesso nei programmi successivi.

Un programma non diventa ordinato per magia. Diventa ordinato quando ogni parte ha una responsabilità chiara. Java non lo farà al posto tuo, perché evidentemente aveva altri impegni.
