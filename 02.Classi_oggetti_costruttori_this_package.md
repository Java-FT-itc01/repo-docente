# 02. Classi, oggetti, costruttori, `this` e package

## Obiettivo

In questa attività studierai gli elementi tecnici minimi per costruire un primo progetto Java orientato agli oggetti.

Gli argomenti sono:

- classe;
- oggetto;
- attributi;
- metodi di istanza;
- costruttore;
- parola chiave `this`;
- più file Java;
- package;
- compilazione ed esecuzione con package.

---

## 1. Classe

Una classe è un modello.

Descrive:

- quali dati possiede un oggetto;
- quali operazioni può eseguire.

Esempio:

```java
public class Libro {
    String titolo;
    String autore;
    double prezzo;
    int numeroPagine;
}
```

Questa classe definisce la struttura di un libro.

Non crea ancora nessun libro.

---

## 2. Oggetto

Un oggetto è una istanza concreta di una classe.

Esempio:

```java
Libro libro1 = new Libro();
```

Qui:

- `Libro` è il tipo;
- `libro1` è la variabile;
- `new Libro()` crea l'oggetto.

---

## 3. Attributi

Gli attributi rappresentano i dati dell'oggetto.

Esempio:

```java
String titolo;
String autore;
double prezzo;
int numeroPagine;
```

Per un oggetto `Libro`, questi attributi descrivono il suo stato.

In questa prima unità non useremo ancora `private`, getter e setter.

Quindi gli attributi saranno lasciati con visibilità di package.

Questa è una scelta didattica temporanea, non il modello finale dell'OOP. Lo ripeteremo, perché altrimenti qualcuno trasformerà questa semplificazione in religione.

---

## 4. Metodi di istanza

Un metodo di istanza è un metodo chiamato su un oggetto specifico.

Esempio:

```java
public void stampaScheda() {
    System.out.println("Titolo: " + titolo);
    System.out.println("Autore: " + autore);
}
```

Chiamata:

```java
libro1.stampaScheda();
```

Il metodo usa gli attributi dell'oggetto su cui viene chiamato.

---

## 5. Costruttore

Il costruttore serve a inizializzare un oggetto quando viene creato.

Esempio:

```java
public Libro(String titolo, String autore, double prezzo, int numeroPagine) {
    this.titolo = titolo;
    this.autore = autore;
    this.prezzo = prezzo;
    this.numeroPagine = numeroPagine;
}
```

Il costruttore ha lo stesso nome della classe:

```java
Libro
```

Non ha tipo di ritorno.

Non si scrive:

```java
public void Libro(...)
```

Sarebbe un metodo, non un costruttore. Java accetta molte stranezze, ma questa no.

---

## 6. Creare un oggetto con il costruttore

Se la classe `Libro` ha questo costruttore:

```java
public Libro(String titolo, String autore, double prezzo, int numeroPagine)
```

allora per creare un oggetto devi passare i valori richiesti:

```java
Libro libro1 = new Libro("Fondamenti di Java", "Mario Rossi", 39.90, 280);
```

I valori passati tra parentesi vengono usati per inizializzare gli attributi.

---

## 7. La parola chiave `this`

Nel costruttore troviamo spesso:

```java
this.titolo = titolo;
```

Il primo `titolo`:

```java
this.titolo
```

indica l'attributo dell'oggetto.

Il secondo `titolo`:

```java
titolo
```

indica il parametro del costruttore.

Quindi:

```java
this.titolo = titolo;
```

significa:

```text
metti nel campo titolo dell'oggetto il valore ricevuto come parametro titolo
```

---

## 8. Errore tipico senza `this`

Osserva:

```java
public Libro(String titolo) {
    titolo = titolo;
}
```

Questa istruzione è inutile.

Il parametro viene assegnato a sé stesso.

L'attributo dell'oggetto non viene valorizzato.

Per questo si usa:

```java
this.titolo = titolo;
```

---

## 9. Due file Java

In un primo progetto OOP è utile separare:

- classe del dominio;
- classe applicativa con `main`.

Esempio:

```text
Libro.java
AppLibro.java
```

`Libro.java` contiene il modello.

`AppLibro.java` contiene il metodo `main` e usa gli oggetti.

Questa separazione è importante.

La classe `Libro` non deve essere trasformata in un contenitore casuale di `main`, input e menu. Sarebbe come mettere il volante nel frigorifero: interessante, ma sbagliato.

---

## 10. Package

Il package serve a organizzare le classi.

Esempio:

```java
package corso.lab11;
```

Questa riga deve essere la prima istruzione significativa del file Java.

Se usi questo package, i file devono stare in una struttura coerente:

```text
src/
  corso/
    lab11/
      Libro.java
      AppLibro.java
```

Il percorso delle cartelle deve riflettere il package.

---

## 11. Struttura consigliata

Per LAB11 useremo questa struttura:

```text
lab11/
  src/
    corso/
      lab11/
        Libro.java
        AppLibro.java
  docs/
    evidence_lab11.md
```

I file `Libro.java` e `AppLibro.java` devono iniziare con:

```java
package corso.lab11;
```

---

## 12. Compilazione con package

Dalla cartella `lab11`, compila così:

```bash
javac -d out src/corso/lab11/Libro.java src/corso/lab11/AppLibro.java
```

Il parametro:

```bash
-d out
```

dice a `javac` di mettere i file compilati nella cartella `out`, rispettando la struttura del package.

Dopo la compilazione avrai una struttura simile:

```text
out/
  corso/
    lab11/
      Libro.class
      AppLibro.class
```

---

## 13. Esecuzione con package

Per eseguire devi usare il nome completo della classe:

```bash
java -cp out corso.lab11.AppLibro
```

Non basta:

```bash
java AppLibro
```

perché la classe appartiene al package:

```text
corso.lab11
```

Il nome completo è quindi:

```text
corso.lab11.AppLibro
```

---

## 14. Errori comuni

### Errore 1 - Cartelle non coerenti con il package

Se il file contiene:

```java
package corso.lab11;
```

ma il file è in una cartella casuale, possono nascere errori e confusione.

Usa:

```text
src/corso/lab11/
```

---

### Errore 2 - Eseguire dalla cartella sbagliata

La compilazione deve essere lanciata dalla root del laboratorio:

```text
lab11
```

Non da dentro:

```text
src/corso/lab11
```

Almeno per questa modalità didattica.

---

### Errore 3 - Dimenticare il nome completo della classe

Sbagliato:

```bash
java -cp out AppLibro
```

Corretto:

```bash
java -cp out corso.lab11.AppLibro
```

---

### Errore 4 - Nome file diverso dalla classe pubblica

Se la classe è:

```java
public class Libro
```

il file deve chiamarsi:

```text
Libro.java
```

---

### Errore 5 - Dimenticare `new`

Sbagliato:

```java
Libro libro1 = Libro("Titolo", "Autore", 10.0, 100);
```

Corretto:

```java
Libro libro1 = new Libro("Titolo", "Autore", 10.0, 100);
```

## 16. Sintesi

Per costruire un primo progetto OOP devi controllare tre livelli:

```text
modello mentale -> codice Java -> struttura delle cartelle
```

Il modello mentale è:

```text
classe -> oggetto -> stato -> comportamento
```

Il codice Java usa:

```text
attributi, costruttori, this, metodi di istanza
```

La struttura del progetto deve rispettare il package.
