# 03. Ordinamento semplice e Big O intuitivo

## Obiettivo

In questa attività studierai un primo algoritmo di ordinamento semplice.

L'obiettivo non è imparare l'algoritmo più efficiente.

L'obiettivo è capire:

- che cosa significa ordinare un array;
- come confrontare coppie di elementi;
- come scambiare due valori;
- perché servono due cicli annidati;
- perché l'ordinamento semplice costa più di una ricerca lineare;
- che cosa significa Big O in modo intuitivo.

---

Prima di iniziare dovresti conoscere:

- array;

## 2. Che cosa significa ordinare

Ordinare un array significa riorganizzare i suoi elementi secondo un criterio.

Esempio iniziale:

```text
[5, 2, 8, 1]
```

Ordinamento crescente:

```text
[1, 2, 5, 8]
```

Ordinamento decrescente:

```text
[8, 5, 2, 1]
```

In questa attività useremo un ordinamento crescente semplice basato su confronti e scambi.

---

## 3. Stampare un array con `Arrays.toString`

Per stampare un array in modo leggibile possiamo usare:

```java
Arrays.toString(array)
```

Serve importare:

```java
import java.util.Arrays;
```

Esempio:

```java
import java.util.Arrays;

public class EsempioArraysToString {
    public static void main(String[] args) {
        int[] numeri = {5, 2, 8, 1};

        System.out.println(Arrays.toString(numeri));
    }
}
```

Output:

```text
[5, 2, 8, 1]
```

---

## 4. Scambio di due valori

Per scambiare due valori dentro un array serve una variabile temporanea.

Esempio:

```java
int temp = numeri[i];
numeri[i] = numeri[j];
numeri[j] = temp;
```

Senza `temp`, uno dei due valori verrebbe perso.

È come cambiare due bicchieri pieni di posto: serve un terzo bicchiere. Perfino Java, ogni tanto, ha bisogno di stoviglie concettuali.

## 6. Ordinamento crescente semplice

L'idea è questa:

- prendo una posizione `i`;
- confronto il valore in `i` con quelli successivi;
- se trovo un valore più piccolo, scambio;
- ripeto per tutte le posizioni.

Codice:

```java
public static void ordinaCrescente(int[] numeri) {
    for (int i = 0; i < numeri.length - 1; i++) {
        for (int j = i + 1; j < numeri.length; j++) {
            if (numeri[i] > numeri[j]) {
                int temp = numeri[i];
                numeri[i] = numeri[j];
                numeri[j] = temp;
            }
        }
    }
}
```

## 7. Programma completo di ordinamento

Crea un file chiamato:

```text
OrdinamentoCrescente.java
```

Inserisci:

```java
import java.util.Arrays;

public class OrdinamentoCrescente {

    public static void ordinaCrescente(int[] numeri) {
        for (int i = 0; i < numeri.length - 1; i++) {
            for (int j = i + 1; j < numeri.length; j++) {
                if (numeri[i] > numeri[j]) {
                    int temp = numeri[i];
                    numeri[i] = numeri[j];
                    numeri[j] = temp;
                }
            }
        }
    }

    public static void main(String[] args) {
        int[] numeri = {5, 2, 8, 1};

        System.out.println("Prima: " + Arrays.toString(numeri));

        ordinaCrescente(numeri);

        System.out.println("Dopo: " + Arrays.toString(numeri));
    }
}
```

Output atteso:

```text
Prima: [5, 2, 8, 1]
Dopo: [1, 2, 5, 8]
```

## 8. Ordinare modifica l'array originale

Il metodo `ordinaCrescente(...)` modifica direttamente l'array ricevuto.

Se vuoi conservare l'array originale, devi creare una copia.

Per farlo puoi usare:

```java
Arrays.copyOf(array, array.length)
```

Esempio:

```java
int[] originale = {5, 2, 8, 1};
int[] copia = Arrays.copyOf(originale, originale.length);

ordinaCrescente(copia);
```

Ora:

- `originale` resta invariato;
- `copia` viene ordinato.

## 9. Esempio con copia

Crea un file chiamato:

```text
OrdinamentoConCopia.java
```

Inserisci:

```java
import java.util.Arrays;

public class OrdinamentoConCopia {

    public static void ordinaCrescente(int[] numeri) {
        for (int i = 0; i < numeri.length - 1; i++) {
            for (int j = i + 1; j < numeri.length; j++) {
                if (numeri[i] > numeri[j]) {
                    int temp = numeri[i];
                    numeri[i] = numeri[j];
                    numeri[j] = temp;
                }
            }
        }
    }

    public static void main(String[] args) {
        int[] originale = {5, 2, 8, 1};
```

```text

Ordinato: [1, 2, 5, 8]
```

## 10. Big O intuitivo

Big O è un modo per ragionare su come cresce il costo di un algoritmo quando aumenta la quantità di dati.

In questa fase non serve formalismo pesante.

Serve capire questa differenza:

| Algoritmo | Idea | Costo intuitivo |
|---|---|---|
| ricerca lineare | controllo gli elementi una volta | circa `n` |
| conteggio | controllo tutti gli elementi una volta | circa `n` |
| minimo/massimo | controllo tutti gli elementi una volta | circa `n` |
| ordinamento semplice | confronto molte coppie | circa `n × n` |

Se l'array ha 10 elementi:

- una scansione fa circa 10 controlli;
- due cicli annidati possono fare molti più confronti.

Se l'array ha 1000 elementi:

- una scansione resta circa 1000 passi;
- un doppio ciclo può avvicinarsi a centinaia di migliaia o milioni di confronti.

Ecco perché non tutti gli algoritmi "che funzionano" sono uguali.

## 11. Contare i confronti dell'ordinamento

Possiamo modificare l'ordinamento per contare i confronti.

Crea un file chiamato:

```text
OrdinamentoContaConfronti.java
```

Inserisci:

```java
import java.util.Arrays;

public class OrdinamentoContaConfronti {

    public static int ordinaCrescente(int[] numeri) {
        int confronti = 0;

        for (int i = 0; i < numeri.length - 1; i++) {
            for (int j = i + 1; j < numeri.length; j++) {
                confronti++;

                if (numeri[i] > numeri[j]) {
                    int temp = numeri[i];
                    numeri[i] = numeri[j];
                    numeri[j] = temp;
                }
            }
        }

        return confronti;
    }

    public static void main(String[] args) {
        int[] numeri = {5, 2, 8, 1};

        int confronti = ordinaCrescente(numeri);

        System.out.println("Array ordinato: " + Arrays.toString(numeri));
        System.out.println("Confronti eseguiti: " + confronti);
    }
}
```

Osserva il numero di confronti.

Poi prova con array più lunghi.

## 12. Confronto con ricerca lineare

La ricerca lineare ha un solo ciclo:

```java
for (int i = 0; i < numeri.length; i++) {
    if (numeri[i] == valore) {
        return i;
    }
}
```

L'ordinamento semplice ha due cicli annidati:

```java
for (int i = 0; i < numeri.length - 1; i++) {
    for (int j = i + 1; j < numeri.length; j++) {
        ...
    }
}
```

Questa differenza strutturale spiega perché il costo cresce in modo diverso.

## 13. Errori comuni

### Errore 1: dimenticare `temp`

Sbagliato:

```java
numeri[i] = numeri[j];
numeri[j] = numeri[i];
```

Dopo la prima assegnazione, il valore originale di `numeri[i]` è perso.

---

### Errore 2: usare limiti sbagliati nei cicli

Corretto:

```java
for (int i = 0; i < numeri.length - 1; i++)
```

e:

```java
for (int j = i + 1; j < numeri.length; j++)
```

---

### Errore 3: ordinare l'array originale quando serve conservarlo

Se vuoi mostrare sia array originale sia array ordinato, crea una copia.

---

### Errore 4: pensare che Big O misuri il tempo esatto

Big O non dice quanti millisecondi impiega il programma.

Aiuta a capire come cresce il lavoro quando cresce la dimensione dell'input.
