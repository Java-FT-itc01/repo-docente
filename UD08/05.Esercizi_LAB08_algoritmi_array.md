# 05. Esercizi LAB08 - Algoritmi su array

## Obiettivo degli esercizi

Questi esercizi servono a consolidare gli algoritmi su array:

- scansione;
- somma;
- media;
- minimo;
- massimo;
- ricerca lineare;
- conteggio;
- posizioni multiple;
- ordinamento;
- ragionamento Big O intuitivo.

Il file è una banca esercizi.

Non è necessario svolgere tutto in aula.

---

## 1. Regole generali

Per tutti gli esercizi:

1. usa un solo file Java per esercizio;
2. usa una sola classe per file;
3. puoi usare più metodi statici nello stesso file;
4. se leggi input numerico, usa `nextLine()` e `Integer.parseInt(...)`;
5. usa `try/catch` per gestire input non validi;
6. usa `array.length`, non `array.length()`;
7. usa `i < array.length`, non `i <= array.length`;
8. se ordini un array ma vuoi conservare l'originale, usa una copia.

# Esercizi base

## Esercizio 1 - Somma e media

### File da creare

```text
Esercizio01_SommaMedia.java
```

### Richiesta

Scrivi un programma che:

- legga la dimensione dell'array;
- legga tutti gli elementi;
- calcoli somma;
- calcoli media.

### Metodi richiesti

```java
public static int leggiIntero(Scanner scanner, String messaggio)
```

```java
public static int leggiInteroPositivo(Scanner scanner, String messaggio)
```

```java
public static int[] leggiArray(Scanner scanner, int dimensione)
```

```java
public static int calcolaSomma(int[] array)
```

```java
public static double calcolaMedia(int[] array)
```

### Test minimo

Input:

```text
5
1
2
3
4
5
```

Output atteso:


## Esercizio 8 - Array già ordinato

### File da creare

```text
Esercizio08_ArrayOrdinato.java
```

### Richiesta

Scrivi un programma che stabilisca se un array è già ordinato in modo crescente.

### Metodo richiesto

```java
public static boolean isOrdinatoCrescente(int[] array)
```

### Suggerimento

Devi controllare ogni coppia adiacente:

```java
if (array[i] > array[i + 1]) {
    return false;
}
```

Se non trovi nessuna coppia fuori ordine, restituisci `true`.

# Domande finali

Rispondi in `docs/evidence_lab08_esercizi.md`.

1. Perché la ricerca lineare può fermarsi al primo valore trovato?
2. Perché il conteggio delle occorrenze deve visitare tutto l'array?
3. Perché minimo e massimo devono partire da `array[0]`?
4. Perché la media richiede un cast a `double`?
5. Perché l'ordinamento semplice usa due cicli annidati?
6. Perché serve una variabile temporanea nello scambio?
7. Perché è utile copiare l'array prima di ordinarlo?
8. Qual è la differenza tra ricerca, conteggio e ordinamento?
9. Quale algoritmo tra quelli svolti ti sembra più costoso?
10. Come spiegheresti Big O intuitivo con parole tue?

---

# Sintesi

Questi esercizi consolidano il passaggio da uso base degli array a uso algoritmico degli array.

Gli array non servono solo a contenere valori.

Servono a costruire ragionamenti:

- cercare;
- contare;
- confrontare;
- ordinare;
- misurare il costo delle operazioni.

Questo è un passaggio fondamentale verso problemi di programmazione più realistici.
