# 01. Dal procedurale agli oggetti

## Obiettivo

In questa attività inizierai il passaggio dalla programmazione procedurale alla programmazione orientata agli oggetti.

Finora hai scritto programmi basati soprattutto su:

- `main`;
- metodi statici;
- variabili;
- array;
- funzioni che ricevono dati come parametri.

Ora inizierai a costruire programmi basati su:

- classi;
- oggetti;
- attributi;
- metodi di istanza;
- costruttori.

Questo è un cambio importante.

Non è solo una nuova sintassi. È un diverso modo di organizzare il programma.

---

## 1. Punto di partenza: il codice procedurale

Nel blocco precedente potevi scrivere un programma come questo:

```java
public class AppProceduraleLibro {

    public static void stampaLibro(String titolo, String autore, double prezzo, int numeroPagine) {
        System.out.println("Titolo: " + titolo);
        System.out.println("Autore: " + autore);
        System.out.println("Prezzo: " + prezzo);
        System.out.println("Numero pagine: " + numeroPagine);
    }

    public static double calcolaPrezzoScontato(double prezzo, double percentualeSconto) {
        return prezzo - prezzo * percentualeSconto / 100;
    }

    public static void main(String[] args) {
        String titolo = "Fondamenti di Java";
        String autore = "Mario Rossi";
        double prezzo = 39.90;
        int numeroPagine = 280;

        stampaLibro(titolo, autore, prezzo, numeroPagine);

        double prezzoScontato = calcolaPrezzoScontato(prezzo, 10);
        System.out.println("Prezzo scontato: " + prezzoScontato);
    }
}
```

Questo codice funziona.

Ma i dati del libro sono sparsi in variabili separate:

```java
String titolo
String autore
double prezzo
int numeroPagine
```

e le operazioni sono metodi statici separati dai dati.

---

## 2. Il problema del procedurale quando il programma cresce

Con un solo libro, il codice è gestibile.

Con due libri, inizi a duplicare variabili:

```java
String titolo1;
String autore1;
double prezzo1;
int pagine1;

String titolo2;
String autore2;
double prezzo2;
int pagine2;
```

Con dieci libri, il programma diventa rapidamente scomodo.

Puoi usare array paralleli, ma anche quelli diventano fragili:

```java
String[] titoli;
String[] autori;
double[] prezzi;
int[] pagine;
```

Ora devi ricordare che l'indice `0` di tutti gli array rappresenta lo stesso libro.

È possibile. Non per questo è una buona architettura.

---

## 3. L'idea della programmazione a oggetti

La programmazione orientata agli oggetti raggruppa:

```text
dati + operazioni sui dati
```

Dentro un'unica struttura chiamata **classe**.

Per esempio, invece di avere variabili sparse:

```java
String titolo;
String autore;
double prezzo;
int numeroPagine;
```

possiamo creare una classe:

```java
public class Libro {
    String titolo;
    String autore;
    double prezzo;
    int numeroPagine;
}
```

Questa classe descrive com'è fatto un libro nel nostro programma.

---

## 4. Classe e oggetto

Una **classe** è un modello.

Un **oggetto** è un'entità concreta creata a partire da quel modello.

Esempio:

```java
public class Libro {
    String titolo;
    String autore;
    double prezzo;
    int numeroPagine;
}
```

Questa classe non crea ancora nessun libro.

Per creare un oggetto serve:

```java
Libro libro1 = new Libro();
```

In questa forma base, `new Libro()` crea un oggetto di tipo `Libro`.

---

## 5. Analogia semplice

Classe:

```text
progetto di una casa
```

Oggetto:

```text
una casa costruita davvero
```

Classe:

```text
modello di libro
```

Oggetto:

```text
un libro specifico con titolo, autore, prezzo e pagine
```


---

## 6. Stato e comportamento

Un oggetto può avere:

- **stato**: i suoi dati;
- **comportamento**: le operazioni che può eseguire.

Per un libro:

### Stato

```text
titolo
autore
prezzo
numeroPagine
```

### Comportamento

```text
stampare la scheda
calcolare il prezzo scontato
verificare se è un libro lungo
```

---

## 7. Classe con attributi e metodi

Esempio:

```java
public class Libro {
    String titolo;
    String autore;
    double prezzo;
    int numeroPagine;

    public void stampaScheda() {
        System.out.println("Titolo: " + titolo);
        System.out.println("Autore: " + autore);
        System.out.println("Prezzo: " + prezzo);
        System.out.println("Numero pagine: " + numeroPagine);
    }
}
```

Gli attributi rappresentano i dati.

Il metodo `stampaScheda()` rappresenta un comportamento dell'oggetto.

---

## 8. Metodo statico e metodo di istanza

Finora hai usato molti metodi statici:

```java
public static int calcolaSomma(int[] numeri)
```

Un metodo statico appartiene alla classe come funzione di servizio.

Un metodo di istanza appartiene a un oggetto specifico.

Esempio:

```java
libro1.stampaScheda();
```

Qui il metodo viene chiamato su un oggetto specifico: `libro1`.

Non scriviamo:

```java
Libro.stampaScheda();
```

perché `stampaScheda()` ha bisogno di sapere quale libro stampare.

---

## 9. Perché gli oggetti rendono il programma più ordinato

Con gli oggetti, dati e comportamenti stanno insieme.

Invece di avere:

```java
stampaLibro(titolo, autore, prezzo, numeroPagine);
```

possiamo scrivere:

```java
libro1.stampaScheda();
```

Il codice diventa più leggibile perché l'operazione è collegata all'oggetto.

---

## 10. Esempio concettuale

Procedurale:

```text
funzione stampaLibro + dati del libro passati come parametri
```

OOP:

```text
oggetto libro + metodo stampaScheda
```

Procedurale:

```java
double sconto = calcolaPrezzoScontato(prezzo, 10);
```

OOP:

```java
double sconto = libro1.calcolaPrezzoScontato(10);
```

Nel secondo caso il prezzo è già dentro l'oggetto.

---

## 11. Cosa non studiamo ancora

In questa prima unità OOP non approfondiamo ancora:

- `private`;
- getter e setter;
- incapsulamento completo;
- ereditarietà;
- polimorfismo;
- interfacce;
- classi astratte;
- collezioni di oggetti.

Questi argomenti arriveranno dopo.

Per ora devi concentrarti su:

```text
classe -> oggetto -> attributi -> metodi -> costruttore
```

## 13. Sintesi

Il passaggio fondamentale è questo:

```text
dal programma centrato sulle funzioni
al programma centrato sugli oggetti
```

La classe descrive un modello.

L'oggetto è una istanza concreta del modello.

Gli attributi descrivono lo stato.

I metodi descrivono il comportamento.

Questo è il punto di ingresso nella programmazione a oggetti.

## Nota didattica - Relazioni tra classi rimandate

In questa unità usiamo oggetti singoli e classi semplici.

È normale chiedersi: “ma quando una classe contiene o usa un'altra classe?”

La risposta breve è:

```text
ci arriveremo, ma non adesso
```

Prima servono basi solide su classe, oggetto, costruttore, `this` e metodi di istanza.

Le relazioni tra classi saranno trattate in UD16, dopo ereditarietà, interfacce, polimorfismo e cast.
