# 04. Esercizi LAB11 - Classi, oggetti e costruttori

## Obiettivo degli esercizi

Questi esercizi servono a consolidare il primo passaggio alla programmazione orientata agli oggetti.

Gli obiettivi sono:

- creare classi semplici;
- definire attributi;
- scrivere costruttori;
- usare `this`;
- creare oggetti con `new`;
- scrivere metodi di istanza;
- separare classe del dominio e classe applicativa;
- usare package e più file Java.

---

## 1. Regole generali

Per ogni esercizio:

1. crea una cartella dedicata;
2. usa un package coerente;
3. crea almeno una classe del dominio;
4. crea almeno una classe applicativa con `main`;
5. usa un costruttore;
6. usa almeno un metodo di istanza che stampa dati;
7. usa almeno un metodo di istanza che restituisce un valore;
8. non usare ancora ereditarietà;
9. non usare ancora collezioni;
10. non usare ancora database.

In questa unità non è obbligatorio usare `private`, getter e setter.

Saranno introdotti successivamente.

---

## 2. Struttura consigliata per ogni esercizio

Esempio per l'esercizio `Prodotto`:

```text
esercizio01-prodotto/
  src/

    evidence.md
```

```java
package corso.lab11.esercizio01;
```

```bash
javac -d out src/corso/lab11/esercizio01/Prodotto.java src/corso/lab11/esercizio01/AppProdotto.java
```

Esecuzione:

```bash
java -cp out corso.lab11.esercizio01.AppProdotto
```

# Esercizi base

## Esercizio 1 - Classe `Prodotto`

### Obiettivo

Rappresentare un prodotto di magazzino.

### Classe dominio

Crea una classe:

```text
Prodotto
```

Attributi:

- `String nome`
- `double prezzo`
- `int quantitaMagazzino`

Costruttore:

```java
public Prodotto(String nome, double prezzo, int quantitaMagazzino)
```

Metodi richiesti:

```java
public void stampaScheda()
```

```java
public double calcolaValoreMagazzino()
```

### Classe applicativa

Crea:

```text
AppProdotto
```

Nel `main`:

- crea almeno due prodotti;
- stampa le schede;
- stampa il valore di magazzino di ciascun prodotto.

# Esercizi intermedi

## Esercizio 4 - Classe `ContoCorrente`

### Obiettivo

Rappresentare un conto corrente semplificato.

Attributi:

- `String intestatario`
- `double saldo`

Metodi richiesti:

```java
public void stampaSaldo()
```

```java
public void versa(double importo)
```

```java
public boolean preleva(double importo)
```

Regole:

- `versa` aggiunge denaro solo se l'importo è positivo;
- `preleva` sottrae denaro solo se l'importo è positivo e se il saldo è sufficiente;
- `preleva` restituisce `true` se l'operazione riesce, `false` altrimenti.

Nel `main`:

- crea un conto;
- stampa il saldo;
- fai almeno un versamento;
- fai almeno due prelievi, uno valido e uno non valido.

## 6. Domande finali

Rispondi nel file di evidenza.

1. Che differenza c'è tra classe e oggetto?
2. Che differenza c'è tra attributo e parametro?
3. A cosa serve il costruttore?
4. Perché si usa `this`?
5. Perché il file deve avere lo stesso nome della classe pubblica?
6. Perché è utile separare classe dominio e classe applicativa?
7. Perché un metodo di istanza viene chiamato su un oggetto?
8. Qual è la differenza tra metodo statico e metodo di istanza?
9. Che cosa significa usare un package coerente con le cartelle?
10. Quale esercizio ti ha aiutato di più a capire gli oggetti?

---

## 7. Sintesi

Questi esercizi consolidano il primo passaggio all'OOP.

Il punto fondamentale è:

```text
una classe descrive un modello, un oggetto rappresenta un caso concreto
```

Il costruttore inizializza l'oggetto.

I metodi di istanza lavorano sui dati dell'oggetto.

Questa base sarà necessaria per affrontare incapsulamento, getter, setter e relazioni tra oggetti.
