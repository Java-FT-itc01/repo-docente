# 04. Esercizi LAB10 - Milestone procedurale integrata

## Obiettivo degli esercizi

Questi esercizi servono a consolidare la costruzione di programmi procedurali più ampi.

Ogni esercizio deve usare:

- un solo file Java;
- una sola classe;
- un solo `main`;
- metodi statici;
- input robusto;
- menu;
- test documentati.

Il file è una banca esercizi.

Non è necessario svolgerli tutti in aula.

---

## 1. Regole generali

Per tutti gli esercizi:

1. usa `Scanner` con `nextLine()`;
2. usa `Integer.parseInt(...)` o `Double.parseDouble(...)` quando serve;
3. gestisci input non validi;
4. usa metodi statici;
5. non scrivere tutto nel `main`;
6. organizza il programma con menu e sottomenu se necessario;
7. controlla lo stato dei dati prima di usarli;
8. documenta i test nel file di evidenza.

---

## 2. File di evidenza

Crea:

```text
docs/evidence_lab10_esercizi.md
```

Per ogni esercizio svolto documenta:

- nome del file;
- obiettivo;
- menu;
- metodi principali;
- compilazione;
- esecuzione;
- test;
- errori incontrati;
- soluzioni adottate.

# Esercizi base

## Esercizio 1 - Toolkit numerico essenziale

### File da creare

```text
Esercizio01_ToolkitNumerico.java
```

### Richiesta

Realizza un programma con menu che permetta di:

```text
1. Inserisci array
2. Stampa array
3. Somma e media
4. Minimo e massimo
5. Cerca valore
0. Esci
```

### Requisiti

- usa un array `int[]`;
- controlla se l'array è disponibile;
- usa input robusto;
- separa le operazioni in metodi statici.

---

## Esercizio 2 - Toolkit testuale essenziale

### File da creare

```text
Esercizio02_ToolkitTestuale.java
```

### Richiesta

Realizza un programma con menu che permetta di:

```text
1. Inserisci stringa
2. Stampa stringa
3. Lunghezza
4. Maiuscolo
5. Minuscolo
6. Conta spazi
7. Verifica parola contenuta
0. Esci
```

### Requisiti

- leggi una stringa non vuota;
- controlla se la stringa è disponibile;
- cerca parole ignorando maiuscole e minuscole;
- usa metodi statici.

---

## Esercizio 3 - Programma doppio numeri e stringhe

### File da creare

```text
Esercizio03_NumeriStringhe.java
```

### Richiesta

Realizza un programma con menu principale:

```text
1. Area numerica
2. Area stringhe
0. Esci
```

L'area numerica deve avere almeno:

- inserimento array;
- somma e media;
- ricerca.

L'area stringhe deve avere almeno:

- inserimento stringa;
- lunghezza;
- conteggio vocali;
- verifica parola presente.

# Esercizi intermedi

## Esercizio 4 - Registro voti procedurale avanzato

### File da creare

```text
Esercizio04_RegistroVoti.java
```

### Menu richiesto

```text
1. Inserisci voti
2. Stampa voti
3. Media, minimo e massimo
4. Conta sufficienti
5. Conta insufficienti
6. Stampa copia ordinata crescente
7. Stampa copia ordinata decrescente
0. Esci
```

### Regole

- voti da `0` a `10`;
- sufficiente se voto `>= 6`;
- non modificare l'array originale quando stampi una copia ordinata.

---

## Esercizio 5 - Analizzatore di testo

### File da creare

```text
Esercizio05_AnalizzatoreTesto.java
```

### Menu richiesto

```text
1. Inserisci frase
2. Lunghezza
3. Conta vocali
4. Conta consonanti
5. Conta spazi
6. Verifica parola
7. Maiuscolo e minuscolo
0. Esci
```

### Requisiti

- la frase deve essere non vuota;
- controlla se la frase è disponibile;
- per consonanti considera solo lettere alfabetiche semplici da `a` a `z`.

---

## Esercizio 6 - Catalogo prodotti procedurale minimale

### File da creare

```text
Esercizio06_CatalogoProdotti.java
```

### Richiesta

Gestisci due array paralleli:

```java
String[] nomi;
double[] prezzi;
```

### Menu richiesto

```text
1. Inserisci prodotti
2. Stampa catalogo
3. Cerca prodotto per nome
4. Prezzo minimo e massimo
5. Prezzo medio
0. Esci
```

### Nota

Gli array paralleli sono accettabili qui solo perché siamo ancora nel blocco procedurale.

Più avanti questo problema sarà più adatto alla programmazione a oggetti.

# Challenge

## Challenge 1 - Toolkit numerico con copie dell'array

### File da creare

```text
Challenge01_ToolkitNumericoCopie.java
```

### Richiesta

Realizza un programma che:

- legga un array;
- mantenga l'array originale;
- produca una copia ordinata crescente;
- produca una copia ordinata decrescente;
- stampi tutte le versioni;
- permetta ricerca e conteggio sull'array originale.

Usa:

```java
Arrays.copyOf(...)
```

---

## Challenge 2 - Ricerca multipla in stringhe

### File da creare

```text
Challenge02_RicercaMultiplaStringhe.java
```

### Richiesta

Realizza un programma che:

- legga una frase;
- permetta di cercare più parole una dopo l'altra;
- termini la ricerca quando l'utente scrive `fine`;
- per ogni parola dica se è presente.

Opzionale:

- conta quante volte compare la parola.

---

## Challenge 3 - Mini suite procedurale completa

Challenge03_MiniSuiteProcedurale.java

# Sintesi

Questi esercizi consolidano la milestone procedurale.

Il punto principale è:

```text
integrare più concetti senza perdere ordine
```

Un programma più grande richiede più disciplina:

- menu chiari;
- stato controllato;
- metodi riusabili;
- test significativi;
- responsabilità separate.

Questo è il ponte verso la programmazione a oggetti.
