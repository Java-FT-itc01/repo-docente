# 02. Menu principale, sottomenu e stato del programma

## Obiettivo

In questa attività costruirai la struttura di un programma con:

- menu principale;
- sottomenu numerico;
- sottomenu stringhe;
- stato mantenuto tra una scelta e l'altra.

Questa struttura sarà usata nel laboratorio completo.

---

## 1. Perché servono i sottomenu

Un programma piccolo può avere un solo menu.

Una milestone integrata, invece, rischia di avere troppe voci.

Esempio di menu troppo lungo:

```text
1. Inserisci array
2. Stampa array
3. Somma
4. Media
5. Minimo
6. Massimo
7. Cerca valore
8. Conta occorrenze
9. Ordina crescente
10. Ordina decrescente
11. Inserisci stringa
12. Lunghezza stringa
13. Conta vocali
14. Conta consonanti
15. Conta spazi
16. Cerca parola
0. Esci
```

Funziona, ma è poco leggibile.

Meglio separare:

```text
1. Area numerica
2. Area stringhe
0. Esci
```

Ogni area avrà il suo sottomenu.

---

## 2. Menu principale

Crea un file chiamato:

```text
EsempioMenuPrincipale.java
```

Inserisci:

```java
import java.util.Scanner;

public class EsempioMenuPrincipale {

    public static void stampaMenuPrincipale() {
        System.out.println();
        System.out.println("===== TOOLKIT PROCEDURALE =====");
        System.out.println("1. Area numerica");
        System.out.println("2. Area stringhe");
        System.out.println("0. Esci");
    }

    public static int leggiIntero(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Integer.parseInt(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un intero valido.");
            }
        }
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int scelta;

        do {
            stampaMenuPrincipale();
            scelta = leggiIntero(scanner, "Scelta: ");

            switch (scelta) {
                case 1:
                    System.out.println("Area numerica selezionata.");
                    break;
                case 2:
                    System.out.println("Area stringhe selezionata.");
                    break;
                case 0:
                    System.out.println("Uscita dal programma.");
                    break;
                default:
                    System.out.println("Scelta non valida.");
            }
        } while (scelta != 0);

        scanner.close();
    }
}
```

Compila ed esegui.

## 3. Aggiungere un sottomenu numerico

Un sottomenu può essere gestito da un metodo dedicato:

```java
public static void gestisciAreaNumerica(Scanner scanner) {
    int scelta;

    do {
        stampaMenuNumerico();
        scelta = leggiIntero(scanner, "Scelta: ");

        switch (scelta) {
            case 1:
                System.out.println("Inserimento array.");
                break;
            case 0:
                System.out.println("Ritorno al menu principale.");
                break;
            default:
                System.out.println("Scelta non valida.");
        }
    } while (scelta != 0);
}
```

Il metodo ha il suo ciclo.

Quando l'utente sceglie `0`, torna al menu principale.

## 4. Sottomenu numerico completo di base

Crea un file chiamato:

```text
EsempioSottomenuNumerico.java
```

Inserisci:

```java
import java.util.Scanner;

public class EsempioSottomenuNumerico {

    public static void stampaMenuPrincipale() {
        System.out.println();
        System.out.println("===== MENU PRINCIPALE =====");
        System.out.println("1. Area numerica");
        System.out.println("0. Esci");
    }

    public static void stampaMenuNumerico() {
        System.out.println();
        System.out.println("----- AREA NUMERICA -----");
        System.out.println("1. Inserisci array");
        System.out.println("2. Stampa array");
        System.out.println("0. Torna al menu principale");
    }

    public static int leggiIntero(Scanner scanner, String messaggio) {
        while (true) {
            System.out.print(messaggio);
            String testo = scanner.nextLine();

            try {
                return Integer.parseInt(testo);
            } catch (NumberFormatException e) {
                System.out.println("Errore: devi inserire un intero valido.");
            }
        }
    }

    public static void gestisciAreaNumerica(Scanner scanner) {
        int scelta;

        do {
            stampaMenuNumerico();
            scelta = leggiIntero(scanner, "Scelta: ");

            switch (scelta) {
                case 1:
                    System.out.println("Qui verrà inserito l'array.");
                    break;
                case 2:
                    System.out.println("Qui verrà stampato l'array.");
                    break;
                case 0:
                    System.out.println("Ritorno al menu principale.");
                    break;
                default:
                    System.out.println("Scelta non valida.");
            }
        } while (scelta != 0);
    }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int scelta;

        do {
            stampaMenuPrincipale();
            scelta = leggiIntero(scanner, "Scelta: ");

            switch (scelta) {
                case 1:
                    gestisciAreaNumerica(scanner);
                    break;
                case 0:
                    System.out.println("Uscita dal programma.");
                    break;
                default:
                    System.out.println("Scelta non valida.");
            }
        } while (scelta != 0);

        scanner.close();
    }
}
```

## 5. Il problema dello stato

Nel programma finale l'area numerica dovrà conservare un array.

All'inizio l'array non esiste:

```java
int[] numeri = null;
```

Quando l'utente inserisce i dati:

```java
numeri = leggiArrayInteri(scanner);
```

Da quel momento l'array è disponibile.

Ma se `gestisciAreaNumerica(...)` riceve `numeri` e lo modifica, deve restituire il nuovo array.

Per questo useremo questa forma:

```java
numeri = gestisciAreaNumerica(scanner, numeri);
```

Il metodo avrà questa intestazione:

```java
public static int[] gestisciAreaNumerica(Scanner scanner, int[] numeri)
```

e alla fine:

```java
return numeri;
```

---

## 6. Stato della stringa

L'area stringhe funziona nello stesso modo.

Nel `main` avremo:

```java
String testo = null;
```

Quando l'utente inserisce una stringa:

```java
testo = leggiStringaNonVuota(scanner, "Inserisci una stringa: ");
```

Per mantenere lo stato, useremo:

```java
testo = gestisciAreaStringhe(scanner, testo);
```

Il metodo sarà:

```java
public static String gestisciAreaStringhe(Scanner scanner, String testo)
```

e alla fine:

```java
return testo;
```

## 7. Controllare lo stato prima di usare i dati

Prima di stampare, analizzare o cercare nell'array, bisogna controllare che esista.

```java
public static boolean arrayDisponibile(int[] numeri) {
    return numeri != null;
}
```

Per la stringa:

```java
public static boolean stringaDisponibile(String testo) {
    return testo != null && !testo.isEmpty();
}
```

Se i dati non sono disponibili, il programma deve mostrare un messaggio.

Esempio:

```java
if (!arrayDisponibile(numeri)) {
    System.out.println("Nessun array disponibile. Prima inserisci i dati.");
    return;
}
```

---

## 8. Perché questo passaggio è importante

Senza controllo dello stato, il programma può generare errori come:

```text
NullPointerException
```

Questo accade quando provi a usare un oggetto o array che vale `null`.

Nel programma finale molte funzioni dipendono da dati già inseriti.

Quindi il controllo dello stato è obbligatorio.

Non è un dettaglio decorativo. È il casco prima di salire sulla moto.

## 10. Sintesi

Una milestone con più aree richiede:

```text
menu principale -> sottomenu -> stato -> operazioni
```

Il punto da ricordare è:

```text
se un sottomenu modifica un dato, deve restituirlo al chiamante
```

Questo sarà fondamentale nel laboratorio completo.
