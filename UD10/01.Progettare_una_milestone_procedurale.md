# 01. Progettare una milestone procedurale

## Obiettivo

In questa attività non inizierai subito dal codice.

Prima analizzerai come progettare una milestone procedurale.

Una milestone non è un esercizio isolato.

È un programma più ampio che integra più argomenti già studiati.

L'obiettivo è imparare a progettare un programma che sia:

- leggibile;
- diviso in metodi;
- testabile;
- abbastanza robusto;
- ancora procedurale.

---

## 1. Che cos'è una milestone

Una milestone è un punto di verifica importante del percorso.

In questo caso la milestone serve a verificare se sai usare insieme:

- variabili;
- tipi primitivi;
- input da tastiera;
- parsing;
- `if`;
- `switch`;
- cicli;
- metodi statici;
- stringhe;
- array;
- ricerca;
- conteggio;
- ordinamento.

Non basta che il programma "funzioni".

Deve anche essere organizzato.

Un programma che funziona ma contiene 400 righe nel `main` non è una milestone: è una stanza piena di cavi, e qualcuno prima o poi inciampa.

---

## 2. Programmazione procedurale strutturata

In questa unità non usiamo ancora la programmazione a oggetti.

Quindi valgono queste regole:

- una sola classe;
- un solo file Java;
- un solo `main`;
- molti metodi statici;
- nessuna classe di dominio;
- nessun package complesso.

Questo però non significa scrivere codice casuale.

Anche un programma procedurale può essere ben strutturato.

---

## 3. Il ruolo del `main`

Il `main` deve essere il coordinatore del programma.

Il `main` dovrebbe:

- creare lo `Scanner`;
- inizializzare lo stato;
- mostrare il menu principale;
- leggere la scelta;
- chiamare il metodo corretto;
- chiudere lo `Scanner` alla fine.

Il `main` non dovrebbe:

- contenere tutti i calcoli;
- contenere tutti i controlli;
- contenere tutta la ricerca;
- contenere tutto l'ordinamento;
- contenere tutte le operazioni sulle stringhe.

Esempio di `main` troppo carico:

```java
public static void main(String[] args) {
    // lettura menu
    // lettura array
    // somma
    // media
    // minimo
    // massimo
    // ricerca
    // ordinamento
    // analisi stringa
    // 300 righe dopo...
}
```

Esempio migliore:

```java
public static void main(String[] args) {
    Scanner scanner = new Scanner(System.in);

    int[] numeri = null;
    String testo = null;

    // ciclo menu principale
    // chiamata ai metodi delle aree

    scanner.close();
}
```

---

## 4. Responsabilità dei metodi

Ogni metodo dovrebbe avere una responsabilità chiara.

Esempi:

```java
stampaMenuPrincipale()
```

stampa il menu principale.

```java
leggiIntero(...)
```

legge un intero in modo robusto.

```java
leggiArrayInteri(...)
```

legge un array di interi.

```java
calcolaSomma(...)
```

calcola la somma.

```java
contaVocali(...)
```

conta le vocali in una stringa.

Il metodo non deve fare troppe cose insieme.

Un buon nome aiuta a capire il programma.

Se il nome del metodo deve diventare:

```java
leggiCalcolaStampaOrdinaEForseCerca(...)
```

forse hai creato un mostro. Tenero, ma pur sempre un mostro.

---

## 5. Mappa dei metodi consigliata

Per la milestone finale useremo una struttura simile a questa.

### Menu

```java
stampaMenuPrincipale()
stampaMenuNumerico()
stampaMenuStringhe()
```

### Input

```java
leggiIntero(Scanner scanner, String messaggio)
leggiInteroPositivo(Scanner scanner, String messaggio)
leggiStringaNonVuota(Scanner scanner, String messaggio)
```

### Area numerica

```java
leggiArrayInteri(Scanner scanner)
stampaArray(int[] numeri)
calcolaSomma(int[] numeri)
calcolaMedia(int[] numeri)
trovaMinimo(int[] numeri)
trovaMassimo(int[] numeri)
cercaValore(int[] numeri, int valore)
contaOccorrenze(int[] numeri, int valore)
ordinaCrescente(int[] numeri)
ordinaDecrescente(int[] numeri)
```

### Area stringhe

```java
stampaStringa(String testo)
stampaLunghezza(String testo)
stampaMaiuscoloMinuscolo(String testo)
contaVocali(String testo)
contaConsonanti(String testo)
contaSpazi(String testo)
verificaParola(Scanner scanner, String testo)
```

### Gestione dei sottomenu

```java
gestisciAreaNumerica(Scanner scanner, int[] numeri)
gestisciAreaStringhe(Scanner scanner, String testo)
```

## 6. Stato del programma

La milestone avrà due dati principali:

```java
int[] numeri = null;
String testo = null;
```

All'inizio:

- l'array non è ancora stato inserito;
- la stringa non è ancora stata inserita.

Per questo i metodi devono controllare se i dati sono disponibili.

Esempio:

```java
public static boolean arrayDisponibile(int[] numeri) {
    return numeri != null;
}
```

E per le stringhe:

```java
public static boolean stringaDisponibile(String testo) {
    return testo != null && !testo.isEmpty();
}
```

## 7. Progettazione prima del codice

Prima di scrivere il programma completo, prepara una mini-progettazione.

Nel file di evidenza scrivi:

1. nome del programma;
2. menu principale;
3. sottomenu numerico;
4. sottomenu stringhe;
5. elenco dei metodi statici previsti;
6. dati mantenuti nello stato del programma;
7. test principali da eseguire.

Questa fase evita di improvvisare.

Improvvisare può essere affascinante nel jazz. Nel codice Java, di solito, produce parentesi graffe disperse e variabili chiamate `x2`.

## 9. Domande di verifica

Rispondi nel file di evidenza.

1. Che cosa distingue una milestone da un esercizio isolato?
2. Perché questa milestone è ancora procedurale?
3. Qual è il ruolo del `main`?
4. Perché conviene usare molti metodi statici?
5. Che cosa significa responsabilità chiara di un metodo?
6. Perché è utile disegnare prima i menu?
7. Quali dati rappresentano lo stato del programma?
8. Perché `int[] numeri = null` è utile in questa milestone?
9. Perché `String testo = null` è utile in questa milestone?
10. Quale rischio nasce se si scrive tutta la logica nel `main`?

---

## 10. Sintesi

Una milestone procedurale non misura solo la conoscenza della sintassi.

Misura la capacità di organizzare un piccolo programma.

Il principio da ricordare è:

```text
il main coordina, i metodi lavorano
```

Questo principio sarà fondamentale anche quando passerai alla programmazione a oggetti.
