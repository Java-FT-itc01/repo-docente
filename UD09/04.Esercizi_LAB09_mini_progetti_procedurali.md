# 04. Esercizi LAB09 - Mini-progetti procedurali con menu

## Obiettivo degli esercizi

Questi esercizi servono a consolidare la costruzione di programmi procedurali con menu.

Ogni esercizio richiede:

- una sola classe;
- un solo `main`;
- metodi statici;
- menu testuale;
- ciclo principale;
- input robusto;
- separazione tra input, logica e output.

Il file è una banca esercizi.

Non è necessario svolgere tutto in aula.

---

## 1. Regole generali

Per tutti gli esercizi:

1. crea un solo file `.java` per esercizio;
2. usa una sola classe;
3. crea un metodo `stampaMenu()`;
4. usa un ciclo principale;
5. usa `switch` per gestire le scelte;
6. usa metodi statici per le operazioni;
7. evita di mettere tutta la logica nel `main`;
8. se usi array, controlla che siano stati acquisiti prima di usarli;
9. usa input robusto con `nextLine()` e conversione;
10. documenta i test nel file di evidenza.

---

## 2. File di evidenza

Crea:

```text
docs/evidence_lab09_esercizi.md
```

Per ogni esercizio svolto documenta:

- nome del file;
- obiettivo;
- opzioni del menu;
- metodi creati;
- comando di compilazione;
- comando di esecuzione;
- almeno due test;
- errori incontrati;
- soluzione adottata.

## Esercizio 8 - Menu con sottomenu

### File da creare

```text
Esercizio08_MenuConSottomenu.java
```

### Richiesta

Crea un programma con menu principale:

```text
1. Operazioni numeriche
2. Operazioni su array
0. Esci
```

Se l'utente sceglie `1`, mostra un sottomenu numerico.

Se sceglie `2`, mostra un sottomenu array.

### Obiettivo

Questo esercizio serve a gestire un programma più strutturato senza introdurre ancora classi multiple.

Attenzione: non trasformare il `main` in una discarica di `switch`. Crea metodi come:

```java
public static void menuNumerico(Scanner scanner)
```

```java
public static void menuArray(Scanner scanner)
```

---

## Challenge 2 - Rubrica minimale procedurale

### File da creare

```text
Challenge02_RubricaProcedurale.java
```

### Richiesta

Realizza una rubrica minimale usando due array paralleli:

```java
String[] nomi;
String[] telefoni;
```

Menu richiesto:

```text
1. Inserisci contatti
2. Stampa contatti
3. Cerca telefono per nome
4. Conta contatti
0. Esci
```

### Nota

Questa è ancora programmazione procedurale.

Gli array paralleli sono accettabili qui solo come esercizio.

Più avanti, con la programmazione a oggetti, questo problema sarà risolto meglio con classi come `Contatto`.

# Sintesi

Questi esercizi consolidano il passaggio da piccoli programmi isolati a mini-progetti procedurali.

Il punto fondamentale è:

```text
un programma più grande richiede organizzazione
```

Anche senza OOP, puoi già scrivere codice più leggibile usando:

- menu;
- cicli principali;
- metodi statici;
- utility riusabili;
- controlli sullo stato;
- separazione delle responsabilità.

Questo è il ponte naturale verso la programmazione a oggetti.
