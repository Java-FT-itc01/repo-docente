# 04. Esercizi LAB13 - Astrazione ed ereditarietà

## Obiettivo degli esercizi

Questi esercizi servono a consolidare:

- astrazione;
- relazione `is-a`;
- superclasse;
- sottoclasse;
- `extends`;
- `super(...)`;
- override;
- `@Override`;
- separazione tra dati comuni e dati specifici;
- uso di `ArrayList` per gestire più oggetti della stessa gerarchia.

Il file è una banca esercizi.

Non è necessario svolgere tutto in aula.

---

## 1. Regole generali

Per ogni esercizio:

1. crea una superclasse;
2. crea almeno due sottoclassi;
3. usa `extends`;
4. usa `super(...)` nel costruttore della sottoclasse;
5. evita di duplicare attributi comuni;
6. usa attributi `private`;
7. usa getter e setter dove servono;
8. usa almeno un metodo ridefinito con `@Override`;
9. crea una classe di test con `main`;
10. usa `ArrayList` quando devi gestire più oggetti;
11. documenta i test nel file di evidenza.

# Esercizi base

## Esercizio 1 - Persone, studenti e docenti

### Obiettivo

Creare una gerarchia semplice con una superclasse `Persona`.

### Superclasse

Crea:

```text
Persona
```

Attributi comuni:

- `String nome`
- `String cognome`
- `int eta`

Metodo:

```java
public String descrivi()
```

### Sottoclassi

Crea:

```text
Studente
Docente
```

`Studente` aggiunge:

- `String matricola`
- `double mediaVoti`

`Docente` aggiunge:

- `String materia`
- `int anniEsperienza`

### Richieste

- usare `extends`;
- usare `super(...)`;
- ridefinire `descrivi()` in entrambe le sottoclassi;
- creare un `ArrayList<Persona>`;
- inserire studenti e docenti;
- stampare tutte le descrizioni con un ciclo.

## 7. Sintesi

Questi esercizi consolidano il passaggio da classi isolate a gerarchie di classi.

Il principio fondamentale è:

```text
la superclasse contiene ciò che è comune, le sottoclassi aggiungono ciò che è specifico
```

Quando gestisci più oggetti della gerarchia, usa:

```java
ArrayList<Superclasse>
```

Così il codice resta coerente con il livello operativo raggiunto in UD12.
