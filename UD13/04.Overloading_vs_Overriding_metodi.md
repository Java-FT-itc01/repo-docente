# 03. Overloading vs Overriding dei metodi

## Obiettivo

In questa sezione distinguiamo due concetti spesso confusi:

- **overloading dei metodi**;
- **overriding dei metodi**.

I nomi sono simili, ma indicano due meccanismi diversi.

L'obiettivo è capire:

- quando un metodo è sovraccaricato;
- quando un metodo è ridefinito;
- quale ruolo hanno firma, parametri, ereditarietà e tipo dell'oggetto;
- perché l'overriding prepara il polimorfismo.

---

## 1. Primo confronto

| Concetto | Domanda guida | Dove avviene |
|---|---|---|
| Overloading | Posso avere più metodi con lo stesso nome ma parametri diversi? | Nella stessa classe o nella gerarchia |
| Overriding | Una sottoclasse può ridefinire un metodo ereditato? | Tra superclasse e sottoclasse |

---

## 2. Overloading: stesso nome, parametri diversi

L'overloading avviene quando in una classe esistono più metodi con lo stesso nome ma con firma diversa.

La firma comprende:

- nome del metodo;
- numero dei parametri;
- tipo dei parametri;
- ordine dei parametri.

Esempio:

```java
public class Calcolatrice {

    public int somma(int a, int b) {
        return a + b;
    }

    public double somma(double a, double b) {
        return a + b;
    }

    public int somma(int a, int b, int c) {
        return a + b + c;
    }
}
```

Il metodo `somma` è sovraccaricato.

---

## 3. Cosa non fa parte della firma

Il tipo di ritorno da solo non basta per fare overloading.

Sbagliato:

```java
public int calcola() {
    return 1;
}

public double calcola() {
    return 1.0;
}
```

Questi due metodi hanno stesso nome e stessi parametri.

Java non può distinguerli solo dal tipo di ritorno.

---

## 4. Overriding: stesso metodo ridefinito

L'overriding avviene quando una sottoclasse ridefinisce un metodo della superclasse.

Superclasse:

```java
public class Immobile {
    public String descrivi() {
        return "Immobile generico";
    }
}
```

Sottoclasse:

```java
public class Appartamento extends Immobile {
    @Override
    public String descrivi() {
        return "Appartamento";
    }
}
```

Qui `Appartamento` ridefinisce `descrivi()`.

---

## 5. Differenza pratica

| Aspetto | Overloading | Overriding |
|---|---|---|
| Nome metodo | uguale | uguale |
| Parametri | diversi | uguali |
| Ereditarietà necessaria | no | sì |
| Annotazione `@Override` | no | sì, consigliata |
| Scopo | offrire varianti dello stesso metodo | specializzare un comportamento |

---

## 6. Esempio completo: overloading

```java
public class AnnuncioFormatter {

    public String formato(String titolo) {
        return titolo;
    }

    public String formato(String titolo, double prezzo) {
        return titolo + " - euro " + prezzo;
    }

    public String formato(String titolo, double prezzo, String zona) {
        return titolo + " - euro " + prezzo + " - zona " + zona;
    }
}
```

Qui `formato` è sovraccaricato.

Il metodo cambia in base ai parametri ricevuti.

---

## 7. Esempio completo: overriding

```java
public class Immobile {
    public String descrivi() {
        return "Immobile generico";
    }
}
```

```java
public class Box extends Immobile {
    @Override
    public String descrivi() {
        return "Box auto";
    }
}
```

Qui `descrivi()` è ridefinito.

La sottoclasse fornisce una versione più specifica.

---

## 8. Uso con `ArrayList`

Con una lista di oggetti, la differenza diventa utile.

```java
ArrayList<Immobile> immobili = new ArrayList<>();
immobili.add(new Appartamento(...));
immobili.add(new Box(...));

for (Immobile immobile : immobili) {
    System.out.println(immobile.descrivi());
}
```

Qui non stiamo scegliendo tra metodi sovraccaricati.

Stiamo chiamando un metodo ridefinito nelle sottoclassi.

Quindi il concetto rilevante è overriding.

Il polimorfismo sarà formalizzato in UD14.

---

## 9. Errore comune

Errore:

```java
@Override
public String descrivi(boolean dettagliato) {
    return "Descrizione dettagliata";
}
```

Se nella superclasse esiste solo:

```java
public String descrivi()
```

allora il metodo con parametro non è overriding.

È overloading.

`@Override` serve proprio a far emergere questo errore.

```java

```


Nel file di evidenza spiega quali metodi sono overloaded e quali sono overridden.

## 12. Sintesi

Overloading:

```text
stesso nome, parametri diversi
```

Overriding:

```text
stesso metodo ereditato, comportamento ridefinito nella sottoclasse
```

Non confonderli è fondamentale per non perdersi nel caos dei metodi Java.
