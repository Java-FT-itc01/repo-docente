# 02. Ereditarietà: `extends`, `super(...)`, override e `@Override`

## Obiettivo

In questa attività vedrai la sintassi Java necessaria per realizzare l'ereditarietà.

Gli argomenti sono:

- `extends`;
- costruttore della superclasse;
- `super(...)`;
- override;
- annotazione `@Override`;
- uso prudente di `private`, `protected`, `public`;
- uso di `ArrayList` nei test con più oggetti.

---

## 1. `extends`

La parola chiave:

```java
extends
```

indica che una classe estende un'altra classe.

Esempio:

```java
public class Appartamento extends Immobile {
    ...
}
```

Significato:

```text
Appartamento è una sottoclasse di Immobile
```

Oppure:

```text
Appartamento è un tipo specifico di Immobile
```

---

## 2. Superclasse

Esempio:

```java
public class Immobile {
    private String indirizzo;
    private double superficieMq;
    private double prezzo;

    public Immobile(String indirizzo, double superficieMq, double prezzo) {
        this.indirizzo = indirizzo;
        this.superficieMq = superficieMq;
        this.prezzo = prezzo;
    }
}
```

`Immobile` contiene dati comuni a più tipi di immobile.

---

## 3. Sottoclasse

Esempio:

```java
public class Appartamento extends Immobile {
    private int piano;
    private int numeroStanze;
}
```

`Appartamento` eredita da `Immobile` e aggiunge dati specifici.

---

## 4. Costruttore della superclasse

Se la superclasse ha un costruttore con parametri, la sottoclasse deve chiamarlo.

Esempio:

```java
public Appartamento(String indirizzo, double superficieMq, double prezzo, int piano, int numeroStanze) {
    super(indirizzo, superficieMq, prezzo);
    this.piano = piano;
    this.numeroStanze = numeroStanze;
}
```

La chiamata:

```java
super(indirizzo, superficieMq, prezzo);
```

inizializza la parte `Immobile` dell'oggetto.

---

## 5. `super(...)` deve essere la prima istruzione

Nel costruttore della sottoclasse, la chiamata a `super(...)` deve essere la prima istruzione.

Corretto:

```java
public Appartamento(...) {
    super(indirizzo, superficieMq, prezzo);
    this.piano = piano;
}
```

Sbagliato:

```java
public Appartamento(...) {
    this.piano = piano;
    super(indirizzo, superficieMq, prezzo);
}
```

Java qui non negozia. Ogni tanto ha dei principi, anche se li comunica con messaggi di errore poco poetici.

---

## 6. Accesso agli attributi della superclasse

Nella superclasse useremo attributi `private`.

Esempio:

```java
private String indirizzo;
```

La sottoclasse non accede direttamente all'attributo.

Usa i getter:

```java
getIndirizzo()
getSuperficieMq()
getPrezzo()
```

Questa scelta protegge l'incapsulamento.

---

## 7. `protected`

Il modificatore `protected` permette l'accesso diretto nelle sottoclassi.

Esempio:

```java
protected String indirizzo;
```

In questa fase useremo `protected` con prudenza.

La regola didattica resta:

```text
attributi private, accesso tramite metodi
```

`protected` esiste, ma non deve diventare la scusa per bucare l'incapsulamento.

---

## 8. Metodo ereditato

La superclasse può definire un metodo comune.

```java
public double prezzoAlMetroQuadro() {
    return prezzo / superficieMq;
}
```

Le sottoclassi possono usarlo senza riscriverlo.

Esempio:

```java
Appartamento appartamento = new Appartamento(...);
System.out.println(appartamento.prezzoAlMetroQuadro());
```

---

## 9. Override

Una sottoclasse può ridefinire un metodo della superclasse.

Nella superclasse:

```java
public String descrivi() {
    return "Immobile generico";
}
```

Nella sottoclasse:

```java
@Override
public String descrivi() {
    return "Appartamento in " + getIndirizzo();
}
```

Questa ridefinizione si chiama overriding.

---

## 10. `@Override`

L'annotazione:

```java
@Override
```

segnala che stai ridefinendo un metodo ereditato.

È utile perché il compilatore controlla la firma.

Se sbagli nome o parametri, Java ti avvisa.

E in quel caso, incredibilmente, ti sta facendo un favore.

---

## 11. Overriding corretto

Per fare overriding, il metodo nella sottoclasse deve avere:

- stesso nome;
- stessi parametri;
- tipo di ritorno compatibile;
- visibilità non più restrittiva.

Esempio corretto:

```java
@Override
public String descrivi() {
    return "Box in " + getIndirizzo();
}
```

---

## 12. Errore tipico: firma diversa

Sbagliato:

```java
@Override
public String descrivi(String prefisso) {
    return prefisso + getIndirizzo();
}
```

Questo non ridefinisce `descrivi()`.

È un altro metodo, perché ha parametri diversi.

Questa è una forma di overloading, non overriding.

---

## 13. Ereditarietà e `ArrayList`

Dopo UD12, quando gestiamo più oggetti useremo spesso `ArrayList`.

Esempio:

```java
import java.util.ArrayList;

ArrayList<Immobile> immobili = new ArrayList<>();
immobili.add(new Appartamento("Via Roma", 80, 160000, 2, 3));
immobili.add(new Box("Via Milano", 20, 30000, false, true));
```

Questo funziona perché:

```text
Appartamento extends Immobile
Box extends Immobile
```

Questa parte anticipa in modo leggero il polimorfismo, che sarà il cuore di UD14.

---

## 14. Mini esempio completo

```java
import java.util.ArrayList;

class Animale {
    public String descrivi() {
        return "Animale generico";
    }
}

class Cane extends Animale {
    @Override
    public String descrivi() {
        return "Cane";
    }
}

class Gatto extends Animale {
    @Override
    public String descrivi() {
        return "Gatto";
    }
}

public class MiniEreditarieta {
    public static void main(String[] args) {
        ArrayList<Animale> animali = new ArrayList<>();
        animali.add(new Cane());
        animali.add(new Gatto());

        for (Animale animale : animali) {
            System.out.println(animale.descrivi());
        }
    }
}
```

## 16. Sintesi

La sintassi minima dell'ereditarietà è:

```java
public class Sottoclasse extends Superclasse {
    public Sottoclasse(...) {
        super(...);
    }

    @Override
    public String descrivi() {
        ...
    }
}
```

Il principio importante è:

```text
la sottoclasse eredita la parte comune e specializza ciò che cambia
```
