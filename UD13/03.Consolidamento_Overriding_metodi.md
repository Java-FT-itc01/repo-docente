# 02.B - Consolidamento: overriding dei metodi in Java

## Obiettivo

Questa sezione consolida il concetto di **overriding dei metodi**, uno dei passaggi fondamentali per comprendere ereditarietà e polimorfismo.

L'obiettivo non è soltanto sapere che esiste `@Override`, ma capire **quando una sottoclasse può ridefinire un comportamento ereditato**.

---

## 1. Scenario intuitivo

Immaginiamo una classe generale `Animale`.

Ogni animale può emettere un verso, ma non tutti gli animali emettono lo stesso verso.

```java
public class Animale {
    public void emettiVerso() {
        System.out.println("L'animale emette un verso");
    }
}
```

Ora creiamo due classi più specifiche:

```java
public class Cane extends Animale {
    @Override
    public void emettiVerso() {
        System.out.println("Il cane abbaia");
    }
}
```

```java
public class Gatto extends Animale {
    @Override
    public void emettiVerso() {
        System.out.println("Il gatto miagola");
    }
}
```

`Cane` e `Gatto` ereditano da `Animale`, ma ridefiniscono il metodo `emettiVerso()`.

---

## 2. Definizione

L'**overriding** avviene quando una sottoclasse ridefinisce un metodo già presente nella superclasse.

Per fare overriding, il metodo nella sottoclasse deve avere:

- stesso nome;
- stessi parametri;
- stesso tipo di ritorno, oppure un tipo compatibile;
- visibilità non più restrittiva rispetto al metodo originale.

---

## 3. Perché è utile

L'overriding permette di dire:

```text
questa operazione esiste per tutti, ma ogni sottoclasse la realizza a modo proprio
```

Esempi:

| Superclasse | Metodo | Sottoclassi |
|---|---|---|
| `Animale` | `emettiVerso()` | `Cane`, `Gatto` |
| `Immobile` | `descrivi()` | `Appartamento`, `Box` |
| `Veicolo` | `descrivi()` | `Auto`, `Moto` |
| `Documento` | `stampa()` | `Fattura`, `Contratto` |

---

## 4. Esempio collegato al laboratorio

Superclasse:

```java
public class Immobile {
    public String descrivi() {
        return "Immobile generico";
    }
}
```

Sottoclasse:

```java
public class Appartamento extends Immobile {
    @Override
    public String descrivi() {
        return "Appartamento";
    }
}
```

Sottoclasse:

```java
public class Box extends Immobile {
    @Override
    public String descrivi() {
        return "Box";
    }
}
```

---

## 5. Overriding e `ArrayList`

Dopo la revisione della UD12, useremo `ArrayList` per gestire più oggetti.

Esempio:

```java
import java.util.ArrayList;

ArrayList<Immobile> immobili = new ArrayList<>();
immobili.add(new Appartamento(...));
immobili.add(new Box(...));

for (Immobile immobile : immobili) {
    System.out.println(immobile.descrivi());
}
```

Il ciclo usa un riferimento generale `Immobile`, ma ogni oggetto può avere una propria versione di `descrivi()`.

In UD13 questo serve a consolidare l'overriding.

In UD14 questo diventerà polimorfismo in modo esplicito.

---

## 6. Perché usare `@Override`

Scrivere:

```java
@Override
```

è una buona pratica.

Permette al compilatore di verificare che stai davvero ridefinendo un metodo.

Esempio di errore:

```java
@Override
public String descrivi(String testo) {
    return testo;
}
```

Se nella superclasse esiste solo:

```java
public String descrivi()
```

allora il metodo con parametro non è overriding.

È un altro metodo.

Il compilatore segnala l'errore grazie a `@Override`.

---

## 7. Regola sulla visibilità

Se nella superclasse il metodo è `public`, nella sottoclasse deve restare `public`.

Sbagliato:

```java
@Override
private String descrivi() {
    return "Appartamento";
}
```

Corretto:

```java
@Override
public String descrivi() {
    return "Appartamento";
}
```

Una sottoclasse non può rendere meno accessibile un metodo ereditato.

---

## 8. Overriding e costruttori

I costruttori non si ridefiniscono con overriding.

Non esiste overriding dei costruttori.

La sottoclasse chiama il costruttore della superclasse con:

```java
super(...);
```

Questo è un altro meccanismo.

## 11. Sintesi

L'overriding permette a una sottoclasse di specializzare un comportamento ereditato.

La forma tipica è:

```java
@Override
public String descrivi() {
    return "Descrizione specifica";
}
```

La frase da ricordare è:

```text
stesso metodo, comportamento specializzato nella sottoclasse
```
