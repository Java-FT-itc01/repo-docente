# 04. Esercizi LAB12 - Incapsulamento, `static`, `final` e wrapper

## Obiettivo degli esercizi

Questi esercizi servono a consolidare:

- attributi `private`;
- getter;
- setter;
- controlli nei setter;
- membri `static`;
- costanti `static final`;
- wrapper class;
- progetti con package e più file Java.

Il file è una banca esercizi.

Non è necessario svolgere tutto in aula.

---

## 1. Regole generali

Per tutti gli esercizi:

1. usa un package coerente con le cartelle;
2. crea almeno una classe dominio;
3. crea una classe applicativa con `main`;
4. usa attributi `private`;
5. usa getter dove ha senso;
6. usa setter solo dove ha senso;
7. inserisci controlli nei setter;
8. usa almeno un metodo di istanza;
9. usa almeno un test con valore valido;
10. usa almeno un test con valore non valido.

---

# Esercizi base

## Esercizio 1 - Classe `Prodotto`

### Richiesta

Crea una classe `Prodotto` con attributi privati:

- `String nome`
- `double prezzo`
- `int quantita`

### Requisiti

- costruttore;
- getter per tutti i campi;
- setter con controlli per `prezzo` e `quantita`;
- setter per `nome` che rifiuti stringhe vuote;
- metodo `calcolaValoreMagazzino()`;
- metodo `stampaScheda()`.

### Test nel `main`

- crea due prodotti;
- modifica prezzo e quantità con valori validi;
- prova un prezzo negativo;
- prova una quantità negativa;
- verifica che i valori non validi non vengano accettati.

---

## Esercizio 2 - Classe `Studente` con contatore statico

### Richiesta

Crea una classe `Studente` con attributi privati:

- `String nome`
- `String cognome`
- `int eta`
- `double mediaVoti`

### Requisiti

- costruttore;
- getter e setter;
- controlli su età e media;
- metodo `promosso()`;
- campo `private static int numeroStudentiCreati`;
- metodo statico `getNumeroStudentiCreati()`.

### Regole

- età maggiore di zero;
- media compresa tra `0` e `10`;
- promosso se media `>= 6`.

---

## Esercizio 3 - Classe `Film` con costante `final`

### Richiesta

Crea una classe `Film` con attributi privati:

- `String titolo`
- `String regista`
- `int durataMinuti`
- `double prezzoBiglietto`

### Requisiti

- costante `public static final double IVA_SPETTACOLO = 0.10`;
- costruttore;
- getter e setter;
- metodo `calcolaPrezzoConIva()`;
- metodo `filmLungo()`.

### Regola

Un film è lungo se dura più di 120 minuti.

# Esercizi intermedi

## Esercizio 4 - Classe `ContoCorrente`

### Richiesta

Crea una classe `ContoCorrente` con attributi privati:

- `String intestatario`
- `double saldo`

### Requisiti

- costruttore;
- getter per intestatario e saldo;
- nessun setter diretto per il saldo;
- metodo `versa(double importo)`;
- metodo `preleva(double importo)`;
- metodo `stampaSituazione()`.

### Regole

- non accettare importi negativi;
- non permettere saldo negativo;
- `preleva(...)` deve restituire `true` se l'operazione riesce;
- `preleva(...)` deve restituire `false` se l'operazione fallisce.

Questo esercizio è importante perché mostra che non tutti i campi devono avere un setter. Sì, il generatore automatico di getter e setter non è il tuo superiore gerarchico.

### Requisiti

---

## 7. Sintesi

Questi esercizi consolidano il passaggio da oggetti semplici a oggetti più robusti.

Il principio fondamentale è:

```text
un oggetto deve proteggere il proprio stato
```

Getter, setter, `static`, `final` e wrapper non sono parole chiave da decorazione.

Sono strumenti per progettare classi più affidabili.
