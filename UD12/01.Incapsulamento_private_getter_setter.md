# 01. Incapsulamento, `private`, getter e setter

## Obiettivo

In questa attività imparerai perché gli attributi di una classe non dovrebbero essere lasciati liberamente accessibili dall'esterno.

Gli obiettivi sono:

- capire il problema degli attributi pubblici;
- introdurre il modificatore `private`;
- usare getter per leggere dati;
- usare setter per modificare dati;
- inserire controlli nei setter;
- capire che non tutti gli attributi devono avere un setter.

---

## 1. Punto di partenza: classe troppo aperta

Nel laboratorio precedente potresti aver scritto classi con attributi accessibili direttamente.

Esempio:

```java
public class Libro {
    public String titolo;
    public double prezzo;
}
```

Questa classe è semplice.

Ma è anche fragile.

Un altro pezzo del programma può fare:

```java
Libro libro = new Libro();
libro.prezzo = -100;
```

Il codice compila.

Ma un prezzo negativo non ha senso.

Il compilatore non controlla il significato dei dati. Purtroppo non è un consulente aziendale, fa solo il compilatore.

---

## 2. Il problema dello stato non valido

Lo stato di un oggetto è l'insieme dei suoi dati.

Per un libro:

```text
titolo
autore
prezzo
numeroPagine
```

Uno stato valido potrebbe essere:

```text
titolo = "Java base"
prezzo = 29.90
numeroPagine = 250
```

Uno stato non valido potrebbe essere:

```text
titolo = ""
prezzo = -10
numeroPagine = 0
```

L'oggetto dovrebbe difendere il proprio stato da valori non validi.

---

## 3. Incapsulamento

Incapsulare significa nascondere i dettagli interni e controllare l'accesso ai dati.

In pratica, in Java spesso significa:

```java
private String titolo;
private double prezzo;
```

e poi:

```java
public String getTitolo()
public void setTitolo(String titolo)
```

Gli attributi non vengono modificati direttamente.

Si passa attraverso metodi controllati.

---

## 4. `private`

Il modificatore:

```java
private
```

rende un attributo accessibile solo dentro la stessa classe.

Esempio:

```java
public class Libro {
    private String titolo;
    private double prezzo;
}
```

Da un'altra classe non puoi più scrivere:

```java
libro.prezzo = -100;
```

Questo è voluto.

Non è una limitazione fastidiosa.

È una protezione.

---

## 5. Getter

Un getter permette di leggere il valore di un attributo.

Esempio:

```java
public String getTitolo() {
    return titolo;
}
```

Uso:

```java
System.out.println(libro.getTitolo());
```

Convenzioni comuni:

| Attributo | Getter |
|---|---|
| `titolo` | `getTitolo()` |
| `prezzo` | `getPrezzo()` |
| `numeroPagine` | `getNumeroPagine()` |
| `disponibile` | `isDisponibile()` oppure `getDisponibile()` |

---

## 6. Setter

Un setter permette di modificare il valore di un attributo.

Esempio ingenuo:

```java
public void setPrezzo(double prezzo) {
    this.prezzo = prezzo;
}
```

Questo setter non controlla nulla.

Permette ancora:

```java
libro.setPrezzo(-100);
```

Quindi è solo un modo più lungo per fare danni.

---

## 7. Setter con controllo

Meglio:

```java
public void setPrezzo(double prezzo) {
    if (prezzo >= 0) {
        this.prezzo = prezzo;
    } else {
        System.out.println("Prezzo non valido.");
    }
}
```

Ora la classe può rifiutare valori non sensati.

Questo è uno dei motivi principali per usare i setter.

---

## 8. Uso di `this`

Nel setter:

```java
public void setPrezzo(double prezzo) {
    this.prezzo = prezzo;
}
```

il nome `prezzo` compare due volte.

```java
this.prezzo
```

indica l'attributo dell'oggetto.

```java
prezzo
```

indica il parametro del metodo.

Quindi:

```java
this.prezzo = prezzo;
```

significa:

```text
assegna all'attributo prezzo dell'oggetto il valore ricevuto come parametro
```

---

## 9. Costruttore e setter

Puoi usare i setter anche nel costruttore.

Esempio:

```java
public Libro(String titolo, double prezzo) {
    setTitolo(titolo);
    setPrezzo(prezzo);
}
```

In questo modo le regole di validazione sono riusate anche durante la creazione dell'oggetto.

Attenzione però: se il setter rifiuta un valore, devi decidere quale valore iniziale usare.

Nel laboratorio useremo valori predefiniti prima dei setter.

---

## 10. Non tutti i setter devono esistere

Non devi creare setter automaticamente per ogni attributo.

Esempio: un conto corrente.

Potresti pensare di scrivere:

```java
public void setSaldo(double saldo)
```

Ma questo permetterebbe a qualunque parte del programma di cambiare direttamente il saldo.

Meglio usare metodi più significativi:

```java
public void versa(double importo)
public boolean preleva(double importo)
```

Questi metodi descrivono operazioni reali e possono controllare le regole.

## 11. Esempio guidato: classe `Prodotto`

Crea un file:

```text
Prodotto.java
```

Inserisci:

```java
public class Prodotto {
    private String nome;
    private double prezzo;
    private int quantita;

    public Prodotto(String nome, double prezzo, int quantita) {
        this.nome = "Prodotto senza nome";
        this.prezzo = 0.0;
        this.quantita = 0;

        setNome(nome);
        setPrezzo(prezzo);
        setQuantita(quantita);
    }

    public String getNome() {
        return nome;
    }

    public void setNome(String nome) {
        if (nome != null && !nome.trim().isEmpty()) {
            this.nome = nome.trim();
        } else {
            System.out.println("Nome non valido.");
        }
    }

    public double getPrezzo() {
        return prezzo;
    }

    public void setPrezzo(double prezzo) {
        if (prezzo >= 0) {
            this.prezzo = prezzo;
        } else {
            System.out.println("Prezzo non valido.");
        }
    }

    public int getQuantita() {
        return quantita;
    }

    public void setQuantita(int quantita) {
        if (quantita >= 0) {
            this.quantita = quantita;
        } else {
            System.out.println("Quantità non valida.");
        }
    }

    public double calcolaValoreMagazzino() {
        return prezzo * quantita;
    }

    public void stampaScheda() {
        System.out.println("Nome: " + nome);
        System.out.println("Prezzo: " + prezzo);
        System.out.println("Quantità: " + quantita);
        System.out.println("Valore magazzino: " + calcolaValoreMagazzino());
    }
}
```

## 12. Classe di test

Crea:

```text
AppProdotto.java
```

Inserisci:

```java
public class AppProdotto {
    public static void main(String[] args) {
        Prodotto prodotto = new Prodotto("Mouse USB", 12.50, 10);

        prodotto.stampaScheda();

        System.out.println();
        System.out.println("Tentativo di prezzo non valido:");
        prodotto.setPrezzo(-20);

        System.out.println("Prezzo attuale: " + prodotto.getPrezzo());

        System.out.println();
        System.out.println("Aggiornamento valido:");
        prodotto.setPrezzo(15.90);
        prodotto.setQuantita(8);

        prodotto.stampaScheda();
    }
}
```

Compila:

```bash
javac Prodotto.java AppProdotto.java
```

Esegui:

```bash
java AppProdotto
```

## 13. Errori comuni

### Errore 1 - Attributi ancora pubblici

Sbagliato:

```java
public double prezzo;
```

Corretto:

```java
private double prezzo;
```

---

### Errore 2 - Setter senza controllo

Debole:

```java
public void setPrezzo(double prezzo) {
    this.prezzo = prezzo;
}
```

Meglio:

```java
public void setPrezzo(double prezzo) {
    if (prezzo >= 0) {
        this.prezzo = prezzo;
    }
}
```

---

### Errore 3 - Getter senza `return`

Sbagliato:

```java
public double getPrezzo() {
    prezzo;
}
```

Corretto:

```java
public double getPrezzo() {
    return prezzo;
}
```

---

### Errore 4 - Usare sempre setter senza ragionare

Non tutti i dati devono essere modificabili liberamente.

Prima di creare un setter chiediti:

```text
ha senso permettere questa modifica dall'esterno?
```

## 15. Sintesi

L'incapsulamento serve a proteggere lo stato degli oggetti.

Il principio fondamentale è:

```text
gli attributi sono privati, l'accesso passa da metodi controllati
```

Getter e setter non sono decorazioni.

Sono strumenti per controllare come un oggetto viene letto e modificato.
