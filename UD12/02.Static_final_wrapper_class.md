# 02. `static`, `final` e wrapper class

## Obiettivo

In questa attività studierai tre concetti spesso incontrati insieme nei programmi Java:

- membri `static`;
- costanti `final`;
- wrapper class come `Integer`, `Double`, `Boolean`.

L'obiettivo è capire il ruolo di ciascun concetto senza confonderli.

---

## 1. Membro di istanza e membro di classe

Un attributo normale appartiene al singolo oggetto.

Esempio:

```java
private String titolo;
```

Ogni oggetto `Libro` ha il proprio titolo.

Un attributo `static` appartiene alla classe.

Esempio:

```java
private static int numeroLibriCreati;
```

Questo valore è condiviso da tutti gli oggetti della classe.

---

## 2. Esempio intuitivo

Se crei tre libri:

```java
Libro libro1 = new Libro(...);
Libro libro2 = new Libro(...);
Libro libro3 = new Libro(...);
```

ogni libro ha il proprio:

```text
titolo
autore
prezzo
numeroPagine
```

Ma il contatore:

```java
numeroLibriCreati
```

è unico per la classe.

Serve a sapere quanti oggetti `Libro` sono stati creati.

---

## 3. Campo `static`

Esempio:

```java
private static int numeroLibriCreati = 0;
```

Nel costruttore:

```java
numeroLibriCreati++;
```

Ogni volta che viene creato un oggetto, il contatore aumenta.

Getter statico:

```java
public static int getNumeroLibriCreati() {
    return numeroLibriCreati;
}
```

Uso:

```java
System.out.println(Libro.getNumeroLibriCreati());
```

Il metodo viene chiamato sulla classe, non su un oggetto specifico.

---

## 4. Errore comune con `static`

Non usare `static` per dati che appartengono al singolo oggetto.

Sbagliato:

```java
private static String titolo;
```

Se `titolo` fosse statico, tutti i libri condividerebbero lo stesso titolo.

Questo non ha senso.

A meno che tu stia creando una biblioteca tristissima con un solo libro copiato male.

---

## 5. `final`

La parola chiave:

```java
final
```

indica che un valore non può essere riassegnato dopo l'inizializzazione.

Per le costanti si usa spesso:

```java
public static final double IVA_LIBRI = 0.04;
```

Convenzione:

- le costanti sono scritte in maiuscolo;
- le parole sono separate da underscore.

Esempi:

```java
public static final int SOGLIA_LIBRO_LUNGO = 300;
public static final double IVA_LIBRI = 0.04;
```

---

## 6. `static final`

Quando scrivi:

```java
public static final double IVA_LIBRI = 0.04;
```

stai dicendo:

- `public`: accessibile dall'esterno;
- `static`: appartiene alla classe;
- `final`: non può essere modificata;
- `double`: tipo del valore;
- `IVA_LIBRI`: nome della costante.

Uso:

```java
System.out.println(Libro.IVA_LIBRI);
```

---

## 7. Esempio con costante

```java
public double calcolaPrezzoConIva() {
    return prezzo + prezzo * IVA_LIBRI;
}
```

Il valore dell'IVA non viene scritto direttamente come numero nel metodo.

Evitiamo:

```java
return prezzo + prezzo * 0.04;
```

Perché un numero scritto così è meno chiaro.

Una costante dà un nome al significato del valore.

---

## 8. Tipi primitivi e wrapper

Java ha tipi primitivi:

| Primitivo | Wrapper |
|---|---|
| `int` | `Integer` |
| `double` | `Double` |
| `boolean` | `Boolean` |
| `char` | `Character` |
| `long` | `Long` |

Un wrapper è una classe che rappresenta un valore primitivo come oggetto.

---

## 9. Perché abbiamo già usato i wrapper

Quando scrivi:

```java
Integer.parseInt("25")
```

stai usando la classe wrapper `Integer`.

Quando scrivi:

```java
Double.parseDouble("12.5")
```

stai usando la classe wrapper `Double`.

Quindi i wrapper non sono una stranezza nuova.

Erano già lì, a lavorare nell'ombra. Come spesso accade in Java.

---

## 10. `parseInt` e `valueOf`

Esempio con `parseInt`:

```java
int numero = Integer.parseInt("25");
```

Restituisce un valore primitivo `int`.

Esempio con `valueOf`:

```java
Integer numero = Integer.valueOf("25");
```

Restituisce un oggetto `Integer`.

Nella maggior parte degli esempi base, userai spesso `parseInt`.

In UD12 puoi vedere anche `valueOf` per capire meglio il collegamento con i wrapper.

---

## 11. Esempio con `Double.valueOf`

```java
Double valore = Double.valueOf("12.5");
double prezzo = valore.doubleValue();
```

Oggi Java gestisce spesso automaticamente il passaggio tra wrapper e primitivo.

Questo meccanismo si chiama boxing/unboxing, ma per ora non serve approfondirlo troppo.

Il punto importante è:

```text
Integer, Double e Boolean sono classi
```

e offrono metodi utili.

---

## 12. `Boolean.valueOf`

Esempio:

```java
Boolean disponibile = Boolean.valueOf("true");
System.out.println(disponibile);
```

Risultato:

```text
true
```

Attenzione:

```java
Boolean.valueOf("ciao")
```

non genera errore.

Restituisce `false`.

Questa differenza rispetto a `Integer.valueOf(...)` è importante.

---

## 13. Esempio completo

Crea un file:

```text
EsempioStaticFinalWrapper.java
```

Inserisci:

```java
public class EsempioStaticFinalWrapper {

    public static final double IVA = 0.22;

    private static int oggettiCreati = 0;

    private double prezzo;

    public EsempioStaticFinalWrapper(double prezzo) {
        this.prezzo = prezzo;
        oggettiCreati++;
    }

    public double prezzoConIva() {
        return prezzo + prezzo * IVA;
    }

    public static int getOggettiCreati() {
        return oggettiCreati;
    }

    public static void main(String[] args) {
        EsempioStaticFinalWrapper a = new EsempioStaticFinalWrapper(10);
        EsempioStaticFinalWrapper b = new EsempioStaticFinalWrapper(20);

        Integer numero = Integer.valueOf("25");
        Double importo = Double.valueOf("12.5");
        Boolean attivo = Boolean.valueOf("true");

        System.out.println("Prezzo A con IVA: " + a.prezzoConIva());
        System.out.println("Prezzo B con IVA: " + b.prezzoConIva());
        System.out.println("Oggetti creati: " + EsempioStaticFinalWrapper.getOggettiCreati());
        System.out.println("Integer: " + numero);
        System.out.println("Double: " + importo);
        System.out.println("Boolean: " + attivo);
    }
}
```

---

## 14. Errori comuni

### Errore 1 - Rendere statico un dato dell'oggetto

Sbagliato:

```java
private static double prezzo;
```

Il prezzo deve appartenere al singolo prodotto, libro o film.

---

### Errore 2 - Modificare una costante

Sbagliato:

```java
IVA_LIBRI = 0.10;
```

Se è `final`, non può essere riassegnata.

---

### Errore 3 - Confondere `parseInt` e `valueOf`

```java
int a = Integer.parseInt("25");
Integer b = Integer.valueOf("25");
```

Il primo restituisce un primitivo.

Il secondo restituisce un wrapper.

---

### Errore 4 - Aspettarsi errore da `Boolean.valueOf`

```java
Boolean.valueOf("ciao")
```

restituisce `false`.

Non genera una `NumberFormatException`.

## 16. Sintesi

`static`, `final` e wrapper hanno ruoli diversi.

```text
static -> appartiene alla classe
final -> non può essere riassegnato
wrapper -> classe che rappresenta un valore primitivo
```

Questi concetti completano il passaggio da oggetti semplici a classi più robuste.
